Advanced Shotgun Metagenomics Analysis: Beginner → Advanced

This page is designed to function as a course, a reference, and a project template for short-read shotgun metagenomics. It emphasizes reproducibility, modern tool choices, assembly-free and assembly-based analysis, genome-resolved metagenomics, and publication-quality visualizations. The code is intentionally verbose and heavily annotated so you can adapt it to a local workstation, HPC cluster, or workflow engine.

Raw FASTQ → figures Read-based + assembly-based MAGs + functions + pathways Annotated Bash + Python HPC / workflow-ready
3 analysis layers

Read-level taxonomy, community functional profiling, and contig/MAG-level reconstruction should be treated as complementary, not competing, analyses.

2 main branches

Assembly-free workflows are fast and robust for community summaries. Assembly-based workflows are slower but essential for MAGs, gene neighborhoods, and novel biology.

1 reproducible pipeline

Use explicit metadata, explicit database versions, and a single workflow engine or shell driver so that every figure can be regenerated later.

1) What this course teaches

Foundations

Shotgun metagenomics asks two broad classes of questions: who is there and what can they do. Advanced analyses add a third class: which genomes, genes, pathways, strains, and mobile elements can be reconstructed.

Read-based layer

Taxonomic composition, approximate abundance, community-wide pathway potential, strain-level marker variation, and quick contamination scans.

Assembly-based layer

Contigs, gene catalogs, differential coverage, co-assembly decisions, and discovery of genes or taxa absent from marker-only databases.

Genome-resolved layer

MAG recovery, completeness/contamination estimation, dereplication, taxonomic placement, metabolism reconstruction, and comparative genomics.

Practical principle: do not rely on a single tool or a single branch of analysis. A strong publication usually triangulates the answer using at least one read-based and one assembly-based view.
Useful notation and formulas Relative abundance of taxon i: p_i = c_i / Σ c_i Shannon diversity: H = - Σ p_i log(p_i) Centered log-ratio (CLR) transformation: clr(x_i) = log(x_i / g(x)) where g(x) is the geometric mean of the sample vector. Bray-Curtis dissimilarity between samples A and B: BC = (Σ |A_i - B_i|) / (Σ (A_i + B_i)) Coverage (rough intuition): coverage ≈ (number of bases mapped to contig) / (contig length)

2) Study design and metadata

Critical

The biggest metagenomics mistakes are usually upstream of computation: inconsistent extraction, poor controls, shallow sequencing, missing metadata, and weak replication. Before running tools, decide what the experimental unit is, what covariates matter, and whether your biological question requires assembly-free summaries, MAG recovery, or both.

  • Minimum metadata table: sample_id, condition, timepoint, subject/site, extraction batch, sequencing lane, host, library type, read length, insert size, total raw reads.
  • Controls: blanks, extraction controls, spike-ins when possible, mock community if you need method benchmarking.
  • Replication: biological replication beats technical replication for most ecological questions.
  • Target depth: depends heavily on complexity; high-diversity soil often needs much deeper sequencing than low-diversity host-associated samples.
Avoid mixing questions: “Which genera differ between groups?” and “Which metabolic pathways differ?” and “Can I recover strain-resolved genomes?” can share input data, but they often require different tool settings, databases, and validation logic.

How to choose between Kraken2 and MetaPhlAn

Use Kraken2 when you want broad read-level assignment, fast screening, custom databases, or contamination checks. Use MetaPhlAn when you want conservative marker-based community profiles and better comparability across samples without classifying every read. Many strong studies run both and report agreement at high taxonomic ranks.

How to choose between MEGAHIT and metaSPAdes

Use MEGAHIT when speed and memory efficiency matter or when you have many samples. Try metaSPAdes when you can afford heavier runtime and want to compare whether contig length or gene recovery improves for your specific dataset.

6) Read QC and host removal

Must do

QC is not just trimming. You need to inspect read count loss, GC shifts, adapter content, low-complexity content, duplication, and whether host subtraction removes a suspiciously large fraction of reads.

Typical decisions

  • adapter trimming
  • quality filtering
  • minimum read length
  • poly-G / platform-specific artifacts
  • host subtraction
  • screening for reagent contaminants

What to report

  • raw reads and retained reads per sample
  • fraction removed as host
  • mean read length after trimming
  • overall duplication / complexity
  • samples excluded and why
Bash: fastp + host removal + MultiQC
#!/usr/bin/env bash
set -euo pipefail

# This script expects:
#   1) config/samples.tsv with columns sample_id, read1, read2
#   2) a Bowtie2 index for the host genome at db/host/host_index
# It writes cleaned FASTQ files, host-removed FASTQ files, and QC reports.

mkdir -p results/qc/fastp results/qc/multiqc data/clean data/host_removed logs

tail -n +2 config/samples.tsv | while IFS=$'\t' read -r sample_id condition timepoint subject batch host read1 read2; do
  echo "[INFO] Processing ${sample_id}"

  # --- fastp performs adapter trimming, quality filtering, and report generation.
  fastp \
    -i "${read1}" \
    -I "${read2}" \
    -o "data/clean/${sample_id}_R1.clean.fastq.gz" \
    -O "data/clean/${sample_id}_R2.clean.fastq.gz" \
    --detect_adapter_for_pe \
    --thread 8 \
    --html "results/qc/fastp/${sample_id}.fastp.html" \
    --json "results/qc/fastp/${sample_id}.fastp.json" \
    > "logs/${sample_id}.fastp.log" 2>&1

  # --- Map cleaned reads to the host. Reads that do NOT align are kept.
  bowtie2 \
    -x db/host/host_index \
    -1 "data/clean/${sample_id}_R1.clean.fastq.gz" \
    -2 "data/clean/${sample_id}_R2.clean.fastq.gz" \
    --very-sensitive \
    --threads 8 \
    --un-conc-gz "data/host_removed/${sample_id}_R%.host_removed.fastq.gz" \
    -S /dev/null \
    > "logs/${sample_id}.host_removal.log" 2>&1
done

# Aggregate all reports
multiqc results/qc/fastp logs -o results/qc/multiqc
Numerical mini-example: host removal fraction

Suppose sample S01 starts with 40 million read pairs. After fastp, 37 million remain. After host subtraction, 12 million remain. Then:

Retention after fastp = 37 / 40 = 92.5% Host-free fraction after cleaning = 12 / 37 = 32.4% Overall retained fraction from raw = 12 / 40 = 30.0%

A low host-free fraction is not automatically bad. In gut biopsy or host tissue samples, this may be expected. But for stool or soil, it could signal a sample prep or reference issue.

7) Taxonomic profiling

Read-based

Taxonomic profiling answers “which organisms are represented and at what approximate abundance?”. In advanced workflows, it is good practice to run at least two methods with different assumptions.

Classifier-style

Kraken2 + Bracken: very fast, database-driven, broad classification, useful for screening, contamination checks, and high-throughput projects.

Marker-style

MetaPhlAn: uses clade-specific markers, often producing conservative and stable abundance estimates across studies.

Bash: Kraken2 + Bracken
#!/usr/bin/env bash
set -euo pipefail

mkdir -p results/taxonomy/kraken2 results/taxonomy/bracken

DB="db/kraken2_standard"
READ_LEN=150

tail -n +2 config/samples.tsv | while IFS=$'\t' read -r sample_id condition timepoint subject batch host read1 read2; do
  kraken2 \
    --db "${DB}" \
    --paired \
    --threads 12 \
    --report "results/taxonomy/kraken2/${sample_id}.kreport.txt" \
    --output "results/taxonomy/kraken2/${sample_id}.kraken.txt" \
    "data/host_removed/${sample_id}_R1.host_removed.fastq.gz" \
    "data/host_removed/${sample_id}_R2.host_removed.fastq.gz"

  # Bracken improves abundance estimation from Kraken classification counts
  bracken \
    -d "${DB}" \
    -i "results/taxonomy/kraken2/${sample_id}.kreport.txt" \
    -o "results/taxonomy/bracken/${sample_id}.species.bracken.tsv" \
    -r "${READ_LEN}" \
    -l S
done
Bash: MetaPhlAn
#!/usr/bin/env bash
set -euo pipefail

mkdir -p results/taxonomy/metaphlan

tail -n +2 config/samples.tsv | while IFS=$'\t' read -r sample_id condition timepoint subject batch host read1 read2; do
  metaphlan \
    data/host_removed/${sample_id}_R1.host_removed.fastq.gz,data/host_removed/${sample_id}_R2.host_removed.fastq.gz \
    --input_type fastq \
    --nproc 12 \
    -o results/taxonomy/metaphlan/${sample_id}.profile.tsv
done
When would you add Kaiju?

Kaiju classifies reads against protein-level references, which can help in some cases where nucleotide-level classifiers miss distant homology. It is worth trying when your dataset is evolutionarily novel, viral-rich, or when you want a complementary classifier with different biases.

8) Assembly and contig statistics

Assembly-based

Assembly collapses reads into longer sequences and makes gene neighborhoods, mobile elements, and genome reconstruction possible. For large projects, compare at least one speed-oriented and one heavier assembler on a subset before scaling up.

Bash: per-sample assembly with MEGAHIT
#!/usr/bin/env bash
set -euo pipefail

mkdir -p results/assembly/megahit

tail -n +2 config/samples.tsv | while IFS=$'\t' read -r sample_id condition timepoint subject batch host read1 read2; do
  megahit \
    -1 "data/host_removed/${sample_id}_R1.host_removed.fastq.gz" \
    -2 "data/host_removed/${sample_id}_R2.host_removed.fastq.gz" \
    -o "results/assembly/megahit/${sample_id}" \
    --min-contig-len 1000 \
    -t 16
done
Bash: alternative metaSPAdes run
#!/usr/bin/env bash
set -euo pipefail

# metaSPAdes is heavier. Use it for a selected subset first.
sample_id=S01

metaspades.py \
  -1 "data/host_removed/${sample_id}_R1.host_removed.fastq.gz" \
  -2 "data/host_removed/${sample_id}_R2.host_removed.fastq.gz" \
  -o "results/assembly/metaspades/${sample_id}" \
  -t 32 \
  -m 250
Do not overinterpret N50 alone. In metagenomics, longer contigs are helpful, but you should also check total assembled bases, gene recovery, mapping rate back to contigs, contamination, and downstream bin quality.
Numerical mini-example: assembly evaluation
Assembly A: total assembled bases = 1.8 Gbp N50 = 5.2 kbp reads mapping back = 84% high-quality MAGs = 34 Assembly B: total assembled bases = 1.6 Gbp N50 = 8.1 kbp reads mapping back = 71% high-quality MAGs = 19 Interpretation: Assembly B has longer contigs, but Assembly A may be better for community recovery. The "best" assembly depends on the biological endpoint.

9) Binning and MAG recovery

Genome-resolved

Binning groups contigs likely belonging to the same genome, usually using sequence composition and differential coverage across samples. A strong modern strategy is to run multiple binners and then integrate them.

Bash: map reads back to contigs and compute depth
#!/usr/bin/env bash
set -euo pipefail

sample_id=S01
assembly="results/assembly/megahit/${sample_id}/final.contigs.fa"

mkdir -p results/mapping/${sample_id} results/bins/${sample_id}

# Build index for contigs
bowtie2-build "${assembly}" "results/mapping/${sample_id}/contigs"

# Map host-removed reads back to assembly
bowtie2 \
  -x "results/mapping/${sample_id}/contigs" \
  -1 "data/host_removed/${sample_id}_R1.host_removed.fastq.gz" \
  -2 "data/host_removed/${sample_id}_R2.host_removed.fastq.gz" \
  -p 16 | samtools sort -@ 8 -o "results/mapping/${sample_id}/${sample_id}.bam"

samtools index "results/mapping/${sample_id}/${sample_id}.bam"

# MetaBAT2 depth file
jgi_summarize_bam_contig_depths \
  --outputDepth "results/bins/${sample_id}/depth.txt" \
  "results/mapping/${sample_id}/${sample_id}.bam"
Bash: MetaBAT2, CONCOCT, and DAS Tool
#!/usr/bin/env bash
set -euo pipefail

sample_id=S01
assembly="results/assembly/megahit/${sample_id}/final.contigs.fa"
depth="results/bins/${sample_id}/depth.txt"

mkdir -p results/bins/${sample_id}/metabat2 \
         results/bins/${sample_id}/concoct \
         results/bins/${sample_id}/dastool

# --- MetaBAT2
metabat2 \
  -i "${assembly}" \
  -a "${depth}" \
  -o "results/bins/${sample_id}/metabat2/bin"

# --- CONCOCT (shown conceptually; in practice prepare coverage table as required)
# concoct --composition_file contigs_10k.fa --coverage_file coverage_table.tsv -b results/bins/${sample_id}/concoct/

# --- DAS Tool integrates outputs from multiple binners.
# Provide a comma-separated list of contigs2bin TSV files and labels.
DAS_Tool \
  -i results/bins/${sample_id}/metabat2_contigs2bin.tsv,results/bins/${sample_id}/concoct_contigs2bin.tsv \
  -l metabat2,concoct \
  -c "${assembly}" \
  -o "results/bins/${sample_id}/dastool/${sample_id}" \
  --write_bins 1 \
  --threads 16
Why multiple samples help binning: differential coverage is informative. If the same genome changes abundance across samples, that pattern helps clustering contigs that belong together.

10) MAG quality, taxonomy, and dereplication

Quality control

Once bins are recovered, estimate completeness and contamination, assign taxonomy, and remove near-duplicate genomes. Otherwise you risk overcounting the same organism many times.

Bash: CheckM2 + GTDB-Tk + dRep
#!/usr/bin/env bash
set -euo pipefail

BINS_DIR="results/bins/S01/dastool/S01_DASTool_bins"
mkdir -p results/mag_qc results/mag_taxonomy results/mag_derep

# --- CheckM2 predicts completeness and contamination
checkm2 predict \
  --threads 24 \
  --input "${BINS_DIR}" \
  --output-directory results/mag_qc

# --- GTDB-Tk classifies bacterial/archaeal genomes
gtdbtk classify_wf \
  --genome_dir "${BINS_DIR}" \
  --out_dir results/mag_taxonomy \
  --cpus 24

# --- dRep dereplicates similar genomes and selects representatives
dRep dereplicate results/mag_derep \
  -g "${BINS_DIR}"/*.fa \
  -p 24

11) Functional profiling and annotation

Functions

Function can be profiled in two complementary ways:

  1. Read-based community function with HUMAnN for gene families and pathways.
  2. Genome-level function by annotating MAGs with DRAM and related tools.
Bash: HUMAnN
#!/usr/bin/env bash
set -euo pipefail

mkdir -p results/function/humann

tail -n +2 config/samples.tsv | while IFS=$'\t' read -r sample_id condition timepoint subject batch host read1 read2; do
  # HUMAnN can take merged or concatenated paired reads.
  # Here we concatenate pairs for simplicity.
  zcat "data/host_removed/${sample_id}_R1.host_removed.fastq.gz" \
       "data/host_removed/${sample_id}_R2.host_removed.fastq.gz" \
    | gzip > "results/function/humann/${sample_id}.merged.fastq.gz"

  humann \
    --input "results/function/humann/${sample_id}.merged.fastq.gz" \
    --output "results/function/humann/${sample_id}" \
    --threads 16
done

Interpretation rule: pathway abundance and pathway completeness are not the same thing. Read-based pathway estimates can be abundant even when no single MAG carries the full pathway, because the pathway can be distributed across taxa.

12) Advanced modules you may add

Optional depth

Strain-level analysis

Use marker-based population workflows such as StrainPhlAn for strain-level comparisons when the target taxa are well represented.

Viral metagenomics

Add viral discovery/curation tools, then validate completeness and contamination of viral contigs separately from bacterial MAG logic.

AMR / virulence

Search assembled genes or MAG proteomes against curated resistance and virulence resources, then interpret in taxonomic context.

  • Co-assembly can help recover low-abundance genomes but may increase chimerism when communities are highly heterogeneous.
  • Long-read or hybrid metagenomics improves contiguity, repeat resolution, plasmids, and viral assemblies when available.
  • anvi'o is especially useful for interactive inspection of coverage patterns, GC, functions, and manual bin refinement.
  • BlobToolKit is useful when decontamination and mixed-origin assembly issues are a major concern.

13) Statistics and normalization

Interpretation

Metagenomics abundance tables are sparse, compositional, and often confounded by library size and host fraction. Raw counts are rarely appropriate for direct comparison.

Common abundance representations Relative abundance: p_i = count_i / total_counts Reads per kilobase (RPK): RPK = raw_reads_assigned / gene_length_kb TPM-style scaling: TPM_i = (RPK_i / Σ RPK) × 10^6 CLR transformation: clr(x_i) = log((x_i + pseudo) / g(x + pseudo))
  • Relative abundance is easy to explain but compositional.
  • CLR-transformed abundances are often better for ordination and machine learning.
  • Presence/absence may be more robust when abundance estimation is noisy.
  • Differential abundance should respect compositionality and sparsity; do not use naive t-tests on raw percentages.
Very common error: interpreting an increase in one taxon as an absolute growth event when the total microbial load or host fraction changed dramatically.

15) Fully annotated Python code for downstream tables and plots

Python

The example below reads a long-format abundance table, keeps the top taxa, makes a stacked bar chart, performs CLR transformation, runs PCA, and builds a heatmap for selected features.

Python: abundance plots, CLR, PCA, heatmap
#!/usr/bin/env python3
"""
Advanced downstream analysis for shotgun metagenomics abundance tables.

Expected input:
    abundance_long.tsv
with columns:
    sample_id, feature, abundance

Optional metadata file:
    metadata.tsv
with columns:
    sample_id, condition, timepoint, batch

This script:
    1) pivots long-format abundances to a matrix
    2) plots top features as stacked bars
    3) applies CLR transformation with a pseudocount
    4) runs PCA
    5) draws a clustered heatmap of the most variable features
"""

from pathlib import Path
import numpy as np
import pandas as pd
import matplotlib.pyplot as plt
from sklearn.decomposition import PCA
from sklearn.preprocessing import StandardScaler

abundance_path = Path("results/tables/abundance_long.tsv")
metadata_path = Path("config/metadata.tsv")
outdir = Path("results/figures/python")
outdir.mkdir(parents=True, exist_ok=True)

ab = pd.read_csv(abundance_path, sep="\t")
meta = pd.read_csv(metadata_path, sep="\t")

mat = ab.pivot_table(index="sample_id", columns="feature", values="abundance", aggfunc="sum", fill_value=0.0)
ordered_samples = [s for s in meta["sample_id"] if s in mat.index]
mat = mat.loc[ordered_samples]

top_n = 15
top_features = mat.sum(axis=0).sort_values(ascending=False).head(top_n).index
mat_top = mat[top_features].copy()
row_sums = mat_top.sum(axis=1).replace(0, np.nan)
rel = mat_top.div(row_sums, axis=0).fillna(0)

fig, ax = plt.subplots(figsize=(12, 5))
bottom = np.zeros(rel.shape[0])
for feature in rel.columns:
    vals = rel[feature].values
    ax.bar(rel.index, vals, bottom=bottom, label=feature)
    bottom += vals
ax.set_ylabel("Relative abundance")

16) Workflow automation, parallelization, and HPC templates

Reproducibility

For more than a few samples, do not manage the project manually. Use Snakemake, Nextflow, or a similarly explicit workflow system. Keep all per-sample tasks parallel and all shared resources versioned.

    input:

        r2=lambda wildcards: [l.split("\t")[7] for l in open("config/samples.tsv").read().strip().splitlines()[1:] if l.startswith(wildcards.sample + "\t")][0]

        r1="data/clean/{sample}_R1.clean.fastq.gz",
        r2="data/clean/{sample}_R2.clean.fastq.gz",

17) Real-world project template

Blueprint

Imagine a study with 48 stool samples collected at two timepoints from control and treated groups. A strong end-to-end plan would look like this:

  1. Run fastp and aggregate QC with MultiQC.
  2. Subtract host reads with Bowtie2.
  3. Generate read-based community profiles with Kraken2+Bracken and MetaPhlAn.
  4. Run HUMAnN for pathway and gene family profiling.
  5. Assemble each sample with MEGAHIT; optionally co-assemble within each condition or subject group.
  6. Map reads back to contigs and compute depth files.
  7. Run MetaBAT2 plus one additional binner, then integrate bins with DAS Tool.
  8. Assess bins with CheckM2, classify with GTDB-Tk, dereplicate with dRep.
  9. Annotate MAGs with DRAM.
  10. Make publication figures: QC, top taxa, PCA, pathway heatmap, MAG quality scatter, selected metabolism summaries.
  11. Write a methods supplement with exact database versions and all thresholds.
Expected outputs: one QC report, one taxonomic abundance matrix, one functional pathway matrix, one assembly table, one MAG quality table, one MAG taxonomy table, one annotation table, and a scripts/workflow directory that regenerates all figures.

18) Troubleshooting checklist

Practical
  • Almost all reads disappear after host removal: check that the Bowtie2 index is correct and that you are not accidentally keeping aligned reads instead of unaligned reads.
  • Kraken2 classifies too many reads as unclassified: confirm database contents and read quality; consider a classifier with different assumptions.
  • Assembly is weak: check library complexity, depth, host fraction, and whether some samples should be co-assembled.
  • Bins are contaminated: use multiple binners, inspect coverage patterns, and refine suspicious bins interactively.
  • Pathway results disagree with MAG annotations: remember community function can be distributed across taxa; gene- or pathway-level presence is not always genome-complete in one MAG.
  • Ordination shows batch instead of biology: revisit metadata, extraction batches, and per-sample host fractions.
Most important reporting habit: write down every database build and threshold at the moment you run the analysis. Reproducing a metagenomics result is often impossible without that information.
