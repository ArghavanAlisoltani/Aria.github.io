1) What is a protein language model?

Core concept

A protein language model (PLM) is a model trained on amino-acid sequences with an objective borrowed from language modeling, such as masked-token prediction or next-token prediction. After seeing many protein sequences, the model learns contextual representations that often encode useful information about motifs, domains, family structure, mutational constraint, and function.

\[ \text{sequence} = (x_1, x_2, \ldots, x_L), \qquad x_i \in V \]
\[ H = [h_1, h_2, \ldots, h_L], \qquad h_i \in \mathbb{R}^d \]
Four common modes of use are: frozen embeddings, head-only tuning, full fine-tuning, and parameter-efficient fine-tuning such as LoRA.

2) Why PLMs matter for bacteria and viruses

Applications

In microbial and viral work, labeled data are often limited, noisy, or expensive to generate. PLMs are powerful because they transfer knowledge from massive unlabeled protein corpora into smaller downstream tasks.

4) Main learning objectives

Theory
\[ \mathcal{L}_{MLM} = -\sum_{i\in M} \log p_\theta(x_i \mid x_{\setminus M}) \]
\[ \mathcal{L}_{AR} = -\sum_{t=1}^L \log p_\theta(x_t \mid x_{
\[ \mathcal{L}_{cls} = -\sum_{c=1}^{C} y_c \log \hat{y}_c \qquad \mathcal{L}_{reg} = \frac{1}{N}\sum_i (y_i-\hat{y}_i)^2 \]

Numerical example: if a masked-site model predicts the correct amino acid with probability 0.70, then the site loss is \(-\log(0.70)=0.357\). If the probability drops to 0.20, the loss becomes \(1.609\).

5) Transformer attention from first principles

Architecture
\[ Q = XW_Q,\qquad K = XW_K,\qquad V = XW_V \]
\[ \mathrm{Attention}(Q,K,V)=\mathrm{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right)V \]

Mini-example: if a residue query aligns strongly with one context key and weakly with another, the softmax weights more strongly toward the first context position. In proteins, that means one residue representation can emphasize motif-relevant or long-range context.

Attention is a weighting mechanism, not automatically a mechanistic biological explanation.

6) Tokenization and amino-acid alphabets

Data representation

Many protein PLMs tokenize one residue at a time. Some expect residues separated by spaces; others do not. Ambiguous or rare residues must be handled consistently.

Token typeMeaningCommon handling
20 canonical amino acidsstandard alphabetkeep as-is
B, Z, Xambiguous tokenskeep or map to X depending on model
U, Orare residuesmodel-dependent
CLS, SEP, PAD, MASKspecial tokensadded by tokenizer
# Standardize protein sequences before tokenization
import re

VALID_AA = set("ACDEFGHIKLMNPQRSTVWYBXZUO")

def clean_protein_sequence(seq: str) -> str:
    seq = re.sub(r"\s+", "", seq.upper())
    seq = seq.replace("*", "").replace("-", "")
    seq = "".join(ch if ch in VALID_AA else "X" for ch in seq)
    return seq

print(clean_protein_sequence("MKT--IIALSYIFCLVFA??A"))
Wrong sequence formatting is one of the easiest ways to sabotage a PLM experiment.

8) Data design for bacteria and viruses

Project design
  • Split by biological unit, not just rows. For bacteria, isolate cluster or lineage is often better than random row split.
  • Avoid homology leakage. Highly similar proteins across train and test produce overoptimistic results.
  • Watch metadata leakage. Site, time, or outbreak information may be strongly confounded with labels.
  • Track truncation and ambiguity. Sequence quality matters.
A good project notebook records: sequence source, label definition, split strategy, homology threshold, and truncation rule.

13) GPU usage for PLMs

Hardware

Memory pressure in PLMs comes from model weights, gradients, optimizer state, and activations. Sequence length matters a lot because transformer attention grows quickly with token count.

\[ \text{memory} \approx \text{weights} + \text{optimizer state} + \text{gradients} + \text{activations} \]
HardwareTypical use
CPU onlydebugging, toy examples, small feature extraction
T4 / 16 GBsmall encoder inference and careful LoRA
L4 / 24 GBmoderate PLM inference and tuning
A100 / H100serious fine-tuning, longer proteins, larger batches
Start with frozen embeddings. Move to LoRA. Use full fine-tuning only when the problem truly needs it.

16) Homology-aware splitting

Critical practice

Random row splits are rarely enough for proteins.

import pandas as pd
from sklearn.model_selection import GroupShuffleSplit

merged = pd.read_csv("merged_labels_sequences.csv")

# First train/test split by cluster_id.

17) Frozen embedding baseline

Strong baseline

This is one of the most important baselines in the entire PLM workflow.

import torch
import numpy as np
import pandas as pd
from transformers import AutoTokenizer, AutoModel
from sklearn.linear_model import LogisticRegression
from sklearn.metrics import roc_auc_score, average_precision_score

MODEL_NAME = "Rostlab/prot_bert"
device = "cuda" if torch.cuda.is_available() else "cpu"

tokenizer = AutoTokenizer.from_pretrained(MODEL_NAME, do_lower_case=False)
model = AutoModel.from_pretrained(MODEL_NAME).to(device)
model.eval()

def format_for_model(seq: str) -> str:
    seq = clean_protein_sequence(seq)
    return " ".join(list(seq))

@torch.no_grad()
def embed_sequences(seqs, batch_size=8, max_length=1024):
    all_vecs = []
    for start in range(0, len(seqs), batch_size):
        batch = [format_for_model(s) for s in seqs[start:start+batch_size]]
        toks = tokenizer(batch, return_tensors="pt", padding=True, truncation=True, max_length=max_length).to(device)
        out = model(**toks)
        hidden = out.last_hidden_state
        mask = toks["attention_mask"].unsqueeze(-1)
        pooled = (hidden * mask).sum(dim=1) / mask.sum(dim=1).clamp(min=1)
        all_vecs.append(pooled.cpu().numpy())
    return np.vstack(all_vecs)

train_df = pd.read_csv("train.csv")
val_df   = pd.read_csv("val.csv")
X_train = embed_sequences(train_df["sequence"].tolist())
X_val   = embed_sequences(val_df["sequence"].tolist())
y_train = train_df["phenotype"].to_numpy()
y_val   = val_df["phenotype"].to_numpy()

clf = LogisticRegression(max_iter=5000, class_weight="balanced")
clf.fit(X_train, y_train)
val_prob = clf.predict_proba(X_val)[:, 1]

print("AUROC:", roc_auc_score(y_val, val_prob))
print("AUPRC:", average_precision_score(y_val, val_prob))
This baseline often tells you whether the protein contains predictive signal before you spend GPU time on full fine-tuning.

18) Head-only tuning and linear probing

Transfer learning

A useful middle ground is to keep the PLM frozen and train only a small classifier/regressor on top of pooled embeddings.

\[ \hat{y} = \sigma\big(W\cdot \mathrm{Pool}(H) + b\big) \]

This is stable, memory-light, and often surprisingly competitive.

19) Full fine-tuning

Training strategy

Full fine-tuning updates all parameters. It can help when labels are informative and the dataset is large enough, but it is more memory-hungry and easier to overfit.

import numpy as np
import pandas as pd
from datasets import Dataset
from transformers import AutoTokenizer, AutoModelForSequenceClassification, DataCollatorWithPadding, Trainer, TrainingArguments
from sklearn.metrics import accuracy_score, f1_score, matthews_corrcoef, roc_auc_score

MODEL_NAME = "Rostlab/prot_bert"
num_labels = 2

tokenizer = AutoTokenizer.from_pretrained(MODEL_NAME, do_lower_case=False)

def prep_text(seq):
    return " ".join(list(clean_protein_sequence(seq)))

train_df = pd.read_csv("train.csv")
val_df   = pd.read_csv("val.csv")

train_ds = Dataset.from_pandas(train_df[["sequence", "phenotype"]].rename(columns={"phenotype": "label"}))
val_ds   = Dataset.from_pandas(val_df[["sequence", "phenotype"]].rename(columns={"phenotype": "label"}))

def tokenize(batch):
    texts = [prep_text(s) for s in batch["sequence"]]
    return tokenizer(texts, truncation=True, max_length=1024)

train_ds = train_ds.map(tokenize, batched=True)
val_ds   = val_ds.map(tokenize, batched=True)

model = AutoModelForSequenceClassification.from_pretrained(MODEL_NAME, num_labels=num_labels)

def compute_metrics(eval_pred):
    logits, labels = eval_pred
    preds = logits.argmax(axis=1)
    out = {
        "acc": accuracy_score(labels, preds),
        "f1": f1_score(labels, preds),
        "mcc": matthews_corrcoef(labels, preds),
    }
    if len(np.unique(labels)) == 2:
        probs = np.exp(logits[:,1]) / np.exp(logits).sum(axis=1)
        out["auroc"] = roc_auc_score(labels, probs)
    return out

args = TrainingArguments(
    output_dir="protbert_full_ft",
    learning_rate=2e-5,
    per_device_train_batch_size=2,
    per_device_eval_batch_size=4,
    num_train_epochs=5,
    weight_decay=0.01,
    eval_strategy="epoch",
    save_strategy="epoch",
    load_best_model_at_end=True,
    metric_for_best_model="mcc",
    report_to="none"
)

trainer = Trainer(
    model=model,
    args=args,
    train_dataset=train_ds,
    eval_dataset=val_ds,
    processing_class=tokenizer,
    data_collator=DataCollatorWithPadding(tokenizer=tokenizer),
    compute_metrics=compute_metrics,
)

trainer.train()
print(trainer.evaluate())

20) LoRA and PEFT

Optimization

LoRA keeps the base model mostly frozen and learns small low-rank adapter matrices instead.

\[ W' = W + \Delta W,\qquad \Delta W = BA \]
\[ A\in\mathbb{R}^{r\times d_{in}},\qquad B\in\mathbb{R}^{d_{out}\times r},\qquad r\ll d \]

Numerical intuition: a 1024×1024 layer has about 1.05 million weights. With LoRA rank 8, the added trainable weights are roughly \(1024\times 8 + 8 \times 1024 = 16{,}384\).

from transformers import AutoTokenizer, AutoModelForSequenceClassification
from peft import LoraConfig, get_peft_model, TaskType

MODEL_NAME = "Rostlab/prot_bert"
tokenizer = AutoTokenizer.from_pretrained(MODEL_NAME, do_lower_case=False)
model = AutoModelForSequenceClassification.from_pretrained(MODEL_NAME, num_labels=2)

peft_config = LoraConfig(
    task_type=TaskType.SEQ_CLS,
    r=8,
    lora_alpha=16,
    lora_dropout=0.1,
    target_modules=["query", "key", "value"]
)
model = get_peft_model(model, peft_config)
model.print_trainable_parameters()
LoRA is often the first tuning strategy worth trying on medium and large protein transformers.

21) QLoRA and quantization

Advanced optimization

QLoRA combines low-rank adapters with quantized base weights to reduce memory usage. This can allow tuning of larger models on smaller GPUs, depending on architecture and hardware support.

22) Mixed precision, BF16/FP16, and gradient accumulation

GPU optimization
\[ \text{effective batch} = B_{micro} \times N_{gpu} \times G_{accum} \]

Mini-example: if each GPU fits micro-batch 2, you have 4 GPUs, and accumulation 8, then the effective batch is \(2\times 4\times 8 = 64\).

import torch
from torch import nn

optimizer = torch.optim.AdamW(model.parameters(), lr=2e-5)
scaler = torch.amp.GradScaler("cuda") if device == "cuda" else None
criterion = nn.CrossEntropyLoss()
grad_accum_steps = 8

model.train()
optimizer.zero_grad()

for step, batch in enumerate(train_loader):
    input_ids = batch["input_ids"].to(device)
    attention_mask = batch["attention_mask"].to(device)
    labels = batch["labels"].to(device)

    if device == "cuda":
        with torch.autocast(device_type="cuda", dtype=torch.bfloat16):
            outputs = model(input_ids=input_ids, attention_mask=attention_mask)
            loss = criterion(outputs.logits, labels) / grad_accum_steps
        scaler.scale(loss).backward()
    else:
        outputs = model(input_ids=input_ids, attention_mask=attention_mask)
        loss = criterion(outputs.logits, labels) / grad_accum_steps
        loss.backward()

    if (step + 1) % grad_accum_steps == 0:
        if device == "cuda":
            scaler.step(optimizer)
            scaler.update()
        else:
            optimizer.step()
        optimizer.zero_grad()

23) Activation checkpointing and long proteins

Memory optimization

Long proteins can be difficult because transformer attention grows quickly with sequence length.

\[ \text{attention cost} \sim O(L^2) \]
  • Use activation checkpointing when full activations do not fit.
  • Use domain-aware cropping or sliding windows only if biologically justified.
  • Be cautious with truncation because it can remove the most important motif or domain.

26) Fully annotated custom training loop

Code 
import torch
from torch import nn
from torch.utils.data import Dataset, DataLoader
from transformers import AutoTokenizer, AutoModel

MODEL_NAME = "Rostlab/prot_bert"
device = "cuda" if torch.cuda.is_available() else "cpu"

tokenizer = AutoTokenizer.from_pretrained(MODEL_NAME, do_lower_case=False)
base_model = AutoModel.from_pretrained(MODEL_NAME)

class ProteinClassifier(nn.Module):
    def __init__(self, encoder, hidden_dim, num_labels=2, dropout=0.2):
        super().__init__()
        self.encoder = encoder
        self.dropout = nn.Dropout(dropout)
        self.classifier = nn.Linear(hidden_dim, num_labels)

    def forward(self, input_ids, attention_mask):
        out = self.encoder(input_ids=input_ids, attention_mask=attention_mask)
        hidden = out.last_hidden_state  # [B, L, d]
        mask = attention_mask.unsqueeze(-1)
        pooled = (hidden * mask).sum(dim=1) / mask.sum(dim=1).clamp(min=1)
        logits = self.classifier(self.dropout(pooled))
        return logits

class ProteinDataset(Dataset):
    def __init__(self, df, max_length=1024):
        self.df = df.reset_index(drop=True)
        self.max_length = max_length

    def __len__(self):
        return len(self.df)

    def __getitem__(self, idx):
        row = self.df.iloc[idx]
        seq = " ".join(list(clean_protein_sequence(row["sequence"])))
        toks = tokenizer(seq, truncation=True, max_length=self.max_length, return_tensors="pt")
        item = {k: v.squeeze(0) for k, v in toks.items()}
        item["labels"] = torch.tensor(int(row["phenotype"]), dtype=torch.long)
        return item

def collate_fn(batch):
    return tokenizer.pad(batch, return_tensors="pt")

train_loader = DataLoader(train_ds, batch_size=4, shuffle=True, collate_fn=collate_fn)
val_loader   = DataLoader(val_ds, batch_size=8, shuffle=False, collate_fn=collate_fn)

hidden_dim = base_model.config.hidden_size
model = ProteinClassifier(base_model, hidden_dim).to(device)
criterion = nn.CrossEntropyLoss()
optimizer = torch.optim.AdamW(model.parameters(), lr=2e-5, weight_decay=0.01)

for epoch in range(5):
    model.train()
    running_loss = 0.0
    for batch in train_loader:
        input_ids = batch["input_ids"].to(device)
        attention_mask = batch["attention_mask"].to(device)
        labels = batch["labels"].to(device)

        optimizer.zero_grad()
        logits = model(input_ids=input_ids, attention_mask=attention_mask)
        loss = criterion(logits, labels)
        loss.backward()
        optimizer.step()
        running_loss += loss.item()

    model.eval()
    correct = 0
    total = 0
    with torch.no_grad():
        for batch in val_loader:
            input_ids = batch["input_ids"].to(device)
            attention_mask = batch["attention_mask"].to(device)
            labels = batch["labels"].to(device)
            logits = model(input_ids=input_ids, attention_mask=attention_mask)
            preds = logits.argmax(dim=1)
            correct += (preds == labels).sum().item()
            total += labels.size(0)

    print(f"Epoch {epoch+1}: train_loss={running_loss/max(1,len(train_loader)):.4f}, val_acc={correct/max(1,total):.4f}")

27) Class imbalance and threshold tuning

Optimization
\[ \mathcal{L}_{weighted} = -\sum_i w_{y_i}\log p(y_i\mid x_i) \]
  • Use class-weighted loss or balanced sampling.
  • Report MCC and AUPRC, not only accuracy.
  • Tune the decision threshold on validation data rather than assuming 0.5 is optimal.

28) Continued pretraining / domain adaptation

Advanced topic

If your downstream sequences come from a narrow distribution, such as a single bacterial protein family or viral clade, you can sometimes improve performance by continuing masked-language-model pretraining on unlabeled in-domain sequences before supervised tuning.

This is especially useful when you have many unlabeled in-domain sequences but relatively few labels.

29) Interpretation and in silico mutagenesis

Interpretation
  • Attention maps can suggest which residues interact in representation space.
  • Integrated gradients or saliency can highlight influential positions.
  • In silico mutagenesis can rank substitutions by change in model score.
AA = list("ACDEFGHIKLMNPQRSTVWY")

def single_mutation_scan(seq, model_score):
    base = model_score(seq)
    rows = []
    for i, wt in enumerate(seq):
        for aa in AA:
            if aa == wt:
                continue
            mutant = seq[:i] + aa + seq[i+1:]
            score = model_score(mutant)
            rows.append({
                "position": i + 1,
                "wt": wt,
                "mut": aa,
                "delta_score": score - base,
            })
    return rows
The strongest interpretation combines model signals with domains, motifs, structure, and known biology.

30) Zero-shot mutation scoring

Advanced topic

Some PLMs can score mutations without supervised labels. These scores are useful as priors for ranking variants, but they are not identical to measured phenotype.

Treat zero-shot scores as a biologically informed prior, not as a substitute for carefully labeled data.

31) Retrieval, clustering, and embedding databases

Practical topic

PLMs are also useful as embedding engines for similarity search, cluster discovery, and annotation transfer.

  • Extract one embedding per protein.
  • Index them with a nearest-neighbor engine.
  • Search new bacterial or viral proteins against your curated embedding database.

32) Real-world end-to-end pipelines

Pipeline design

Bacterial pipeline

Collect sequences and labels → clean and deduplicate → cluster-aware split → frozen embedding baseline per protein → multi-protein comparison → LoRA on best model → mutation scan and structural interpretation.

Viral pipeline

Curate sequences and metadata → build time-aware or lineage-aware splits → compare frozen embeddings, head tuning, and LoRA → evaluate calibration and confusion matrix → interpret top positions in domain context.
