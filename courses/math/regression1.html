2) Core regression concepts: loss, likelihood, link

Foundation

Loss

Defines “badness” of predictions. Squared loss punishes large errors heavily; absolute loss is more robust.

Likelihood

Probabilistic view: assume a distribution for \(y|x\). Minimizing negative log-likelihood gives a loss.

Link

In GLMs, relate predictors to mean response: \(g(\mathbb{E}[y|x]) = X\beta\).

Formula explained: generic supervised regression objective
\[ \hat{\beta}=\arg\min_{\beta}\ \sum_{i=1}^n L\Big(y_i,\ f(\mathbf{x}_i;\beta)\Big) + \lambda \Omega(\beta) \]
  • \(f(\mathbf{x}_i;\beta)\) is the model (e.g., \(f=\mathbf{x}_i^T\beta\) for linear regression).
  • \(L(y,\hat{y})\) is the loss (e.g., \((y-\hat{y})^2\), logistic cross-entropy, Poisson deviance).
  • \(\Omega(\beta)\) is the penalty (e.g., \(\|\beta\|_2^2\) or \(\|\beta\|_1\)).
  • \(\lambda\ge 0\) controls the tradeoff between fitting data (first term) and model simplicity (second term).
If \(\lambda=0\), you’re only fitting the training data. If \(\lambda\) is large, you force \(\beta\) to be small/sparse even if training error increases.
Why “minimizing squared error” corresponds to Gaussian noise

Assume \(y_i = \mathbf{x}_i^T\beta + \varepsilon_i\) and \(\varepsilon_i\sim \mathcal{N}(0,\sigma^2)\). The likelihood is:

\[ p(y_i|\mathbf{x}_i,\beta)=\frac{1}{\sqrt{2\pi\sigma^2}}\exp\left(-\frac{(y_i-\mathbf{x}_i^T\beta)^2}{2\sigma^2}\right) \]

Taking \(-\log\) and summing over \(i\) gives a constant + \(\sum (y_i-\mathbf{x}_i^T\beta)^2\). Minimizing it is OLS.

3) Regularization (L1 vs L2): detailed formula meaning

Concept
Formula explained: ridge vs lasso penalties
\[ \text{Ridge: }\ \min_{\beta}\ \|y-X\beta\|_2^2 + \lambda\|\beta\|_2^2 \] \[ \text{Lasso: }\ \min_{\beta}\ \|y-X\beta\|_2^2 + \lambda\|\beta\|_1 \]
  • \(\|y-X\beta\|_2^2 = \sum_{i=1}^n (y_i-\hat{y}_i)^2\) is the total squared error.
  • \(\|\beta\|_2^2 = \sum_{j=1}^p \beta_j^2\): large coefficients cost a lot, so ridge shrinks all coefficients toward 0.
  • \(\|\beta\|_1 = \sum_{j=1}^p |\beta_j|\): linear cost encourages many coefficients to become exactly 0 → sparsity.
A good mental model: Ridge spreads weight across correlated features; Lasso tends to select a subset.
Constraint view (why L1 leads to zeros)

Penalized form is equivalent to a constrained form:

\[ \min \|y-X\beta\|_2^2\ \text{s.t.}\ \|\beta\|_2^2\le t \quad (\text{ridge}) \] \[ \min \|y-X\beta\|_2^2\ \text{s.t.}\ \|\beta\|_1\le t \quad (\text{lasso}) \]

In coefficient space, the L1 “ball” has corners → the optimum often hits a corner → some \(\beta_j=0\).

How to choose λ (most common practice)
  • Use cross-validation to pick \(\lambda\) that minimizes validation error (MSE, deviance, etc.).
  • If you report performance, use nested CV to avoid optimistic bias.
  • Standardize features before ridge/lasso/elastic net (important!).

4) OLS Linear Regression

Baseline
\[ y = X\beta + \varepsilon \]
Formula explained: \(y=X\beta+\varepsilon\)
  • \(X\beta\) is the model’s “systematic” part: prediction from features.
  • \(\varepsilon\) is the “random” part: noise + unmodeled effects.
  • For sample \(i\): \(y_i = \mathbf{x}_i^T\beta + \varepsilon_i\).
\[ \hat{\beta}_{OLS} = \arg\min_{\beta}\ \|y-X\beta\|_2^2 \quad\Rightarrow\quad \hat{\beta}_{OLS} = (X^TX)^{-1}X^Ty \]
Formula explained: OLS objective and solution
  • \(\|y-X\beta\|_2^2 = \sum_i (y_i-\hat{y}_i)^2\) is the sum of squared residuals.
  • The solution \((X^TX)^{-1}X^Ty\) exists when \(X^TX\) is invertible (features not perfectly collinear).
  • Geometric meaning: \(\hat{y}=X\hat{\beta}\) is the orthogonal projection of \(y\) onto the column space of \(X\).
In practice, numerical solvers avoid explicitly computing \((X^TX)^{-1}\); they use QR/SVD for stability.

5) Polynomial Regression

Nonlinear in x, linear in β
\[ \hat{y}=\beta_0+\beta_1 x+\beta_2 x^2+\cdots+\beta_d x^d \]
Formula explained: why it’s still “linear regression”
  • Even though \(x^2,x^3,\dots\) appear, the model is linear in parameters \(\beta_j\).
  • Define a transformed feature vector \(\phi(x)=(1,x,x^2,\dots,x^d)\). Then \(\hat{y}=\phi(x)^T\beta\).
  • So you can still fit with OLS/ridge/lasso—just on the transformed design matrix \(\Phi\).
Python + plot (degree 1 vs 5)
High-degree polynomial features can explode in magnitude; scaling + regularization is typically necessary.

6) Ridge Regression (L2)

Shrinkage
\[ \hat{\beta}_{ridge}=\arg\min_{\beta}\ \|y-X\beta\|_2^2 + \lambda\|\beta\|_2^2 \quad\Rightarrow\quad \hat{\beta}=(X^TX+\lambda I)^{-1}X^Ty \]
Formula explained: what each term is doing
  • \(\|y-X\beta\|_2^2\): fit the data (reduce residual error).
  • \(\lambda\|\beta\|_2^2\): penalize large coefficients; larger \(\lambda\) → more shrinkage.
  • \(I\): identity matrix \((p\times p)\). Adding \(\lambda I\) makes \(X^TX+\lambda I\) invertible even when \(X^TX\) is singular.
  • Geometric meaning: ridge changes the projection so it is less sensitive to directions with low data support (small singular values).
Ridge is especially useful when predictors are correlated (common in omics).

7) Lasso Regression (L1)

Sparsity
\[ \hat{\beta}_{lasso}=\arg\min_{\beta}\ \|y-X\beta\|_2^2 + \lambda\|\beta\|_1 \]
Formula explained: why L1 produces zeros
  • \(\|\beta\|_1=\sum_j |\beta_j|\) has a “kink” at 0 (not differentiable there).
  • Optimization can land exactly at \(\beta_j=0\) because moving away from 0 immediately increases penalty linearly.
  • Result: many coefficients become 0 → sparse model → interpretable feature set.
With correlated predictors, lasso may select one feature and drop others even if they are all relevant. Elastic Net can help.
Python + plot (coefficients shrink to zero)

8) Elastic Net (L1 + L2)

Group-friendly sparsity
\[ \hat{\beta}=\arg\min_{\beta}\ \|y-X\beta\|_2^2 + \lambda\left(\alpha\|\beta\|_1+(1-\alpha)\|\beta\|_2^2\right) \]
Formula explained: what α does
  • \(\alpha\in[0,1]\) controls the mixture: higher \(\alpha\) → more lasso-like (sparser); lower \(\alpha\) → more ridge-like (more shrinkage, less sparsity).
  • \(\lambda\) controls overall penalty strength (how strongly coefficients are forced toward 0).
  • Elastic Net often keeps groups of correlated features instead of choosing just one.
In scikit-learn, the parameterization is consistent with the equation above (alpha = overall strength, l1_ratio = mixing).
Python + plot (compare Lasso vs ElasticNet)

9) Logistic Regression (binary outcomes)

GLM (Bernoulli)
\[ p_i = P(y_i=1|\mathbf{x}_i)=\sigma(\mathbf{x}_i^T\beta),\quad \sigma(z)=\frac{1}{1+e^{-z}} \]
Formula explained: sigmoid and log-odds
  • \(z_i=\mathbf{x}_i^T\beta\) is a linear score (can be any real number).
  • \(\sigma(z)\) maps \(\mathbb{R}\to(0,1)\) so we get a valid probability.
  • The inverse link is logit: \(\log\frac{p}{1-p}=z\). So logistic regression is “linear in log-odds”.
\[ \mathcal{L}(\beta)= -\sum_{i=1}^n \left[y_i\log p_i + (1-y_i)\log(1-p_i)\right] + \lambda\Omega(\beta) \]
Formula explained: cross-entropy loss
  • The bracket term is the negative log-likelihood of a Bernoulli outcome.
  • If \(y_i=1\), the loss is \(-\log p_i\): it punishes low predicted probability for the true class.
  • If \(y_i=0\), the loss is \(-\log(1-p_i)\): it punishes high predicted probability for class 1 when true class is 0.
  • Regularization \(\lambda\Omega(\beta)\) prevents extreme coefficients (which can overfit).
Python + plot (probability curve)

10) Poisson Regression (counts)

GLM (Poisson)
\[ y_i \sim \mathrm{Poisson}(\mu_i),\quad \log \mu_i = \mathbf{x}_i^T\beta \]
Formula explained: Poisson mean model
  • \(\mu_i = \mathbb{E}[y_i|\mathbf{x}_i]\) is the expected count for sample \(i\).
  • The log link \(\log \mu_i = \mathbf{x}_i^T\beta\) ensures \(\mu_i>0\) automatically (since \(\mu_i=e^{\mathbf{x}_i^T\beta}\)).
  • Interpretation: a one-unit increase in feature \(x_{ij}\) multiplies \(\mu_i\) by \(e^{\beta_j}\) (holding others fixed).
This multiplicative interpretation is why Poisson regression is popular for rates and counts.
Python + plot (fitted mean curve)
Poisson assumes variance ≈ mean. If variance is much larger (overdispersion), use Negative Binomial.

11) Negative Binomial Regression (overdispersed counts)

Counts + dispersion
\[ y_i \sim \mathrm{NB}(\mu_i, \theta),\quad \log \mu_i = \mathbf{x}_i^T\beta \]
Formula explained: what NB adds beyond Poisson
  • Poisson: \(\mathrm{Var}(y_i|\mathbf{x}_i)=\mu_i\).
  • Negative Binomial allows: \(\mathrm{Var}(y_i|\mathbf{x}_i)=\mu_i + \alpha \mu_i^2\) (one common parameterization).
  • \(\alpha\) (or \(\theta\)) is the dispersion: larger dispersion → more variability than Poisson.
  • In RNA-seq, biological variability makes NB a much better model than Poisson.
If you’ve used DESeq2/edgeR in R: their core count model is NB with gene-specific dispersion.
Python + plot (Poisson vs NB fit)

12) Robust Regression (Huber)

Outlier-resistant loss
\[ r_i = y_i - \mathbf{x}_i^T\beta \]
Formula explained: residual
  • \(r_i\) is how far off your prediction is for sample \(i\).
  • OLS minimizes \(\sum r_i^2\), which makes large residuals extremely influential.
\[ L_\delta(r)= \begin{cases} \frac{1}{2}r^2 & |r|\le \delta\\ \delta(|r|-\frac{1}{2}\delta) & |r|>\delta \end{cases} \]
Formula explained: why Huber is robust
  • For small errors (\(|r|\le \delta\)), it behaves like squared loss → smooth and efficient when noise is Gaussian.
  • For large errors (\(|r|>\delta\)), it becomes linear in \(|r|\) → outliers don’t dominate.
  • \(\delta\) is the “switch point”: smaller \(\delta\) → more robust (more linear region).

13) Quantile Regression (median / conditional quantiles)

Not just the mean
\[ \rho_\tau(u)=u(\tau-\mathbb{1}[u<0]) \]
Formula explained: pinball (check) loss
  • \(u = y - \mathbf{x}^T\beta\) is the residual.
  • \(\mathbb{1}[u<0]\) is an indicator: 1 if residual is negative, else 0.
  • If \(u\ge 0\): \(\rho_\tau(u)=\tau u\) (slope \(\tau\)).
  • If \(u<0\): \(\rho_\tau(u)=(\tau-1)u\) (slope \(\tau-1\), which is negative).
  • This asymmetry forces the fitted line so that a fraction \(\tau\) of points lie below it (approximately).
For \(\tau=0.5\) (median), pinball loss becomes proportional to absolute error: minimizing it gives the conditional median.
\[ \hat{\beta}_\tau = \arg\min_{\beta}\sum_{i=1}^n \rho_\tau\left(y_i-\mathbf{x}_i^T\beta\right) \]
Formula explained: quantile regression objective
  • Instead of minimizing squared errors (mean), it minimizes an asymmetric absolute-like loss (quantile).
  • Useful when noise is skewed or variance changes with \(x\) (heteroscedasticity).

14) Practical checklist (bioinformatics)

Use this every time
A) Match the model to the outcome
  • Continuous: OLS/Ridge/Lasso/ElasticNet/Huber/Quantile
  • Binary: Logistic
  • Counts: Poisson; if overdispersed → Negative Binomial
B) Always standardize features for regularization
  • Ridge/lasso compare coefficient magnitudes. If one feature has huge units, it will dominate.
  • Use z-scoring: \(x'=(x-\mu)/\sigma\) within the training fold.
C) Control confounding and leakage
  • Include batch, sex, age, ancestry PCs, library size/purity, platform, etc.
  • Split data by patient or study batch if needed (avoid leakage across folds).
  • Do feature selection inside CV folds (never on the whole dataset).
In omics: you almost always want regularization (or dimension reduction) + careful splitting.
