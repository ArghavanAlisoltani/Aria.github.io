10 Regression Methods + Regularization

This version adds more conceptual explanation (when to use each method, what assumptions it makes, what can go wrong, and how regularization changes the behavior). Each method includes math, numerical intuition, Python code, and plots.

High-level mental model

Every regression method answers: “What function \(f(x)\) best predicts \(y\)?” The differences are about (1) the assumed noise distribution, (2) the link function (GLMs), (3) the loss function (robust / quantile), and (4) whether we penalize complexity (regularization).

1) Core concepts: loss, noise, link, and bias–variance

Conceptual foundation

Loss function

Regression is “choose parameters to minimize a loss”. Different losses imply different robustness properties (e.g., squared loss vs absolute loss).

Noise model / likelihood

In probabilistic regression, you assume a distribution for \(y\mid x\): Gaussian (OLS), Bernoulli (logistic), Poisson/NB (counts).

Link function (GLM)

GLMs model \(g(\mathbb{E}[y|x]) = X\beta\). For Poisson, \(g=\log\); for logistic, \(g=\mathrm{logit}\).

Bias–variance tradeoff

More flexible models fit training data better (low bias) but can be unstable (high variance). Regularization reduces variance by limiting complexity.

Why squared loss corresponds to Gaussian noise (important intuition)

If you assume \(y_i = \mathbf{x}_i^T\beta + \epsilon_i\) and \(\epsilon_i \sim \mathcal{N}(0,\sigma^2)\), then maximizing likelihood is equivalent to minimizing the sum of squared residuals:

\[ -\log p(y|X,\beta) \propto \sum_i (y_i-\mathbf{x}_i^T\beta)^2 \]
What can go wrong in real bioinformatics regression?
  • Confounding: batch effects, ancestry/structure, library size, processing pipelines
  • Collinearity: correlated predictors (e.g., genes in same pathway)
  • Non-Gaussian outcomes: counts, zero-inflation, heavy tails, heteroscedasticity
  • p≫n: many more features than samples; regularization becomes essential

2) Regularization: deep intuition (geometry + effects)

Conceptual

Regularization adds a penalty to discourage overly complex models. It is not “just a trick”—it encodes a belief (a prior) that parameters should be small or sparse.

Penalty geometry: why L1 creates sparsity

Consider minimizing SSE subject to a constraint: \(\|\beta\|_2 \le t\) (ridge) vs \(\|\beta\|_1 \le t\) (lasso). In 2D, L2 constraint is a circle; L1 is a diamond with corners on axes. The optimum often hits a corner → one coefficient becomes exactly 0.

\[ \text{Ridge: } \min \|y-X\beta\|_2^2 \ \text{s.t.}\ \|\beta\|_2^2 \le t \quad\Longleftrightarrow\quad \min \|y-X\beta\|_2^2 + \lambda\|\beta\|_2^2 \] \[ \text{Lasso: } \min \|y-X\beta\|_2^2 \ \text{s.t.}\ \|\beta\|_1 \le t \quad\Longleftrightarrow\quad \min \|y-X\beta\|_2^2 + \lambda\|\beta\|_1 \]
Bayesian view (optional but powerful)
  • Ridge (L2) ↔ Gaussian prior on \(\beta\): \(\beta \sim \mathcal{N}(0,\tau^2I)\)
  • Lasso (L1) ↔ Laplace prior on \(\beta\) (sharper peak at 0 → sparsity)
How to choose λ (most common practice)
  • Cross-validation (CV) to minimize validation error
  • Nested CV if you want unbiased performance estimates
  • For GLMs: tune via deviance / log-likelihood on held-out data
Rule of thumb: in high-dimensional genomics (\(p\gg n\)), you almost always need regularization (or dimensionality reduction) to generalize.

3) OLS Linear Regression

Baseline

When to use

Continuous outcomes with roughly constant variance; good baseline and interpretable coefficients.

Assumptions (ideal)

Linearity, independent errors, constant variance, Gaussian errors (for p-values/CI).

\[ y = X\beta + \varepsilon,\quad \hat{\beta}_{OLS} = (X^TX)^{-1}X^Ty \]
Python + plot (fit line)
What can go wrong?
  • Outliers can dominate the fit (see Huber section).
  • Nonlinear patterns: OLS underfits (see polynomial features).
  • Collinearity increases coefficient variance (ridge helps).

4) Polynomial Regression (basis expansion)

Flexible but can overfit

Why it works

Turns nonlinear relationships into a linear model in an expanded feature space.

Risk

High degree → wiggly curve → low training error but poor generalization.

\[ \hat{y}=\beta_0+\beta_1 x+\beta_2 x^2+\cdots+\beta_d x^d \]

5) Ridge Regression (L2)

Stabilizes coefficients

When to use

Many correlated features; you want to keep all predictors but shrink them.

Effect

Reduces coefficient variance; improves predictions; rarely sets coefficients exactly to 0.

\[ \hat{\beta}_{ridge}=\arg\min_{\beta}\ \|y-X\beta\|_2^2 + \lambda\|\beta\|_2^2,\quad \hat{\beta}=(X^TX+\lambda I)^{-1}X^Ty \]

6) Lasso Regression (L1)

Sparse model

When to use

You suspect only a small subset of features matter (sparse signal), and you want interpretability.

Effect

Many coefficients become exactly zero (feature selection-like behavior).

\[ \hat{\beta}_{lasso}=\arg\min_{\beta}\ \|y-X\beta\|_2^2 + \lambda\|\beta\|_1 \]
Python + plot (coefficients shrink to zero)
Important caveat in genomics

If many features are highly correlated (genes in pathways), lasso may pick one arbitrarily. Elastic Net often behaves better for grouped signals.

7) Elastic Net (L1 + L2)

Group-friendly sparsity

When to use

High-dimensional data with correlated predictors (common in omics).

Effect

Combines shrinkage (L2) + sparsity (L1); more stable than pure lasso with correlations.

\[ \hat{\beta}=\arg\min_{\beta}\ \|y-X\beta\|_2^2 + \lambda\left(\alpha\|\beta\|_1+(1-\alpha)\|\beta\|_2^2\right) \]

8) Logistic Regression (binary outcomes)

GLM

Interpretation

Linear model on log-odds: \(\log\frac{p}{1-p}=\mathbf{x}^T\beta\). Coefficients change odds multiplicatively.

When to use

Binary outcomes: response/non-response, case/control, mutation present/absent.

\[ P(y=1|\mathbf{x})=\sigma(\mathbf{x}^T\beta),\quad \sigma(z)=\frac{1}{1+e^{-z}} \] \[ \text{Loss}=-\sum_i \left[y_i\log p_i + (1-y_i)\log(1-p_i)\right] + \lambda\Omega(\beta) \]
Python + plot (probability curve)

9) Poisson Regression (counts)

GLM

When to use

Counts like mutation counts per sample, reads mapping to a feature, events per unit time.

Assumption

Mean ≈ variance (often violated in RNA-seq; then use Negative Binomial).

\[ y_i\sim \mathrm{Poisson}(\mu_i),\quad \log \mu_i = \mathbf{x}_i^T\beta \]
Python + plot (fitted mean curve)

10) Negative Binomial Regression (overdispersed counts)

Counts

Why it matters in bioinformatics

RNA-seq and metagenomic counts are overdispersed: variance > mean due to biological heterogeneity.

Interpretation

Like Poisson but with an extra dispersion parameter to allow more variability.

\[ y_i \sim \mathrm{NB}(\mu_i, \theta),\quad \log \mu_i = \mathbf{x}_i^T\beta \]

11) Robust Regression (Huber)

Outlier-resistant

Concept

Squared loss explodes for large residuals. Huber changes to linear growth, reducing outlier influence.

When to use

Outliers, heavy-tailed noise, measurement artifacts, occasional failed samples.

\[ L_\delta(r)= \begin{cases} \frac{1}{2}r^2 & |r|\le \delta\\ \delta(|r|-\frac{1}{2}\delta) & |r|>\delta \end{cases} \]
Python + plot (OLS vs Huber with outliers)

12) Quantile Regression (median / conditional quantiles)

Robust + informative

What it gives you

Not just mean response—median or other quantiles (e.g., 0.1 and 0.9) to model uncertainty bands.

When to use

Heteroscedastic data (variance changes with x), skewed noise, interest in tails.

\[ \min_{\beta}\sum_i \rho_\tau(y_i-\mathbf{x}_i^T\beta), \quad \rho_\tau(u)=u(\tau-\mathbb{1}[u<0]) \]

13) Practical checklist (bioinformatics)

Do this before modeling
1) Decide what your outcome distribution is
  • Continuous (normalized expression, log counts): OLS/Ridge/Lasso/ElasticNet/Huber/Quantile
  • Binary (case/control): Logistic
  • Counts (mutations, reads): Poisson or Negative Binomial
2) Control confounding and leakage
  • Include batch, sex, age, tumor purity, ancestry PCs, sequencing platform, etc.
  • Split train/test by patient (not by sample) when there are repeated measures.
  • Do scaling/normalization inside CV folds to avoid leakage.
3) Always tune λ via cross-validation
  • Use nested CV for performance estimates; simple CV for model selection.
  • Report: chosen λ, coefficient stability, and performance variance across folds.
In omics, the model that “looks best” on training often fails in validation — regularization + careful splits are the fix.
