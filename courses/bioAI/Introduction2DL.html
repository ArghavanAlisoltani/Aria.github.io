1) Perceptron

Foundations

The perceptron is the simplest neural classifier: it computes a weighted sum of inputs and applies a step function. It is useful for understanding how neural networks turn features into decisions, but it can only separate linearly separable data.

Main formulas
\[ z = \mathbf{w}^\top \mathbf{x} + b \]
\[ \hat{y} = \mathbb{1}(z \ge 0) \]
\[ \mathbf{w} \leftarrow \mathbf{w} + \eta (y-\hat{y}) \mathbf{x}, \quad b \leftarrow b + \eta (y-\hat{y}) \]
Numerical example

Let \(\mathbf{x}=[2,1]\), \(\mathbf{w}=[0.6,-0.2]\), and \(b=-0.1\). Then \(z = 0.6(2) + (-0.2)(1) - 0.1 = 0.9\). Since \(z \ge 0\), the perceptron predicts class \(1\). If the true label were \(0\), the update would push the weights in the opposite direction.

Annotated Python code 
# Perceptron forward pass and one manual update
import torch

# One sample with two features
x = torch.tensor([2.0, 1.0])

# Current weights and bias
w = torch.tensor([0.6, -0.2])
b = torch.tensor(-0.1)

# Linear score z = w^T x + b
z = torch.dot(w, x) + b

# Step activation: predict 1 if z >= 0 else 0
y_hat = (z >= 0).float()
print("score z:", z.item(), "prediction:", int(y_hat.item()))

# Suppose the correct label is 0
y_true = torch.tensor(0.0)
eta = 0.1

# Perceptron error term
error = y_true - y_hat

# Weight update: move weights against the mistaken prediction
w = w + eta * error * x
b = b + eta * error

print("updated w:", w)
print("updated b:", b.item())
Key takeaways
  • Best for: teaching the core idea of weighted decisions.
  • Limitation: cannot solve XOR without adding hidden layers.

2) Multilayer Perceptron (MLP)

Feedforward

An MLP stacks linear layers and nonlinear activations. This allows the network to learn curved decision boundaries and richer feature interactions than a single perceptron.

Main formulas
\[ \mathbf{h} = \phi(\mathbf{W}_1 \mathbf{x} + \mathbf{b}_1) \]
\[ \mathbf{o} = \mathbf{W}_2 \mathbf{h} + \mathbf{b}_2 \]
\[ \hat{\mathbf{y}} = \mathrm{softmax}(\mathbf{o}) \]
Numerical example

Take \(\mathbf{x}=[1,2]\). Suppose \[ \mathbf{W}_1 = egin{bmatrix}1 & -1\0.5 & 0.5\end{bmatrix},\; \mathbf{b}_1=egin{bmatrix}0\0\end{bmatrix}. \] Then \(\mathbf{W}_1\mathbf{x}=[-1,1.5]\), so after ReLU the hidden state is \([0,1.5]\). A second layer can convert this hidden representation into class scores.

Annotated Python code 
# Small MLP for a single forward pass
import torch
import torch.nn as nn

# Two inputs -> four hidden units -> three output classes
model = nn.Sequential(
    nn.Linear(2, 4),   # affine transform
    nn.ReLU(),         # nonlinearity
    nn.Linear(4, 3)    # class logits
)

# One sample
x = torch.tensor([[1.0, 2.0]])

# Forward pass returns raw scores (logits)
logits = model(x)

# Convert logits to probabilities
probs = torch.softmax(logits, dim=1)

print("logits:", logits)
print("probabilities:", probs)
Key takeaways
  • Common for tabular features, baseline classifiers, and small regression tasks.
  • The depth and hidden width determine model capacity.

3) Convolutional Neural Network (CNN)

Vision

CNNs exploit spatial locality by sliding learnable filters across an input. The same filter is reused at many positions, which drastically reduces parameters compared with a fully connected image model.

Main formulas
\[ (\mathbf{X} * \mathbf{K})_{i,j} = \sum_{m}\sum_{n} X_{i+m,j+n} K_{m,n} \]
\[ \mathbf{H} = \phi(\mathbf{X} * \mathbf{K} + b) \]
Numerical example

For a \(3 imes 3\) image patch \[ \mathbf{X}=egin{bmatrix}1&2&0\0&1&3\2&1&1\end{bmatrix} \] and a \(2 imes 2\) filter \[ \mathbf{K}=egin{bmatrix}1&0\-1&1\end{bmatrix}, \] the top-left output value is \(1(1)+2(0)+0(-1)+1(1)=2\).

Annotated Python code 
# Minimal CNN example on one 8x8 grayscale image
import torch
import torch.nn as nn

# Batch size 1, channels 1, height 8, width 8
x = torch.randn(1, 1, 8, 8)

cnn = nn.Sequential(
    nn.Conv2d(1, 4, kernel_size=3, padding=1),  # 4 feature maps
    nn.ReLU(),
    nn.MaxPool2d(2),                             # 8x8 -> 4x4
    nn.Conv2d(4, 8, kernel_size=3, padding=1),
    nn.ReLU()
)

y = cnn(x)
print("input shape :", x.shape)
print("output shape:", y.shape)
Key takeaways
  • Great for images, spectrograms, and any grid-like data.
  • Key ideas: local receptive fields, weight sharing, pooling.

4) LeNet-5

Classic CNN

LeNet-5 is one of the earliest successful CNN architectures. It established the now-familiar pattern of convolution, pooling, and fully connected classification.

Main formulas
\[ 32\times 32 \rightarrow \text{Conv} \rightarrow \text{Pool} \rightarrow \text{Conv} \rightarrow \text{Pool} \rightarrow \text{FC} \rightarrow \text{FC} \]
\[ \hat{\mathbf{y}} = \mathrm{softmax}(\mathbf{W}\mathbf{h} + \mathbf{b}) \]
Numerical example

A \(32 imes 32\) grayscale image passed through a \(5 imes 5\) convolution with 6 filters yields 6 feature maps. After pooling, the spatial size shrinks while the number of channels grows, which lets later layers use broader context.

Annotated Python code 
# LeNet-style network for digit-like images
import torch
import torch.nn as nn

class LeNet(nn.Module):
    def __init__(self, n_classes=10):
        super().__init__()
        self.features = nn.Sequential(
            nn.Conv2d(1, 6, kernel_size=5),  # 32 -> 28
            nn.Tanh(),
            nn.AvgPool2d(2),                 # 28 -> 14
            nn.Conv2d(6, 16, kernel_size=5), # 14 -> 10
            nn.Tanh(),
            nn.AvgPool2d(2)                  # 10 -> 5
        )
        self.classifier = nn.Sequential(
            nn.Flatten(),
            nn.Linear(16 * 5 * 5, 120),
            nn.Tanh(),
            nn.Linear(120, 84),
            nn.Tanh(),
            nn.Linear(84, n_classes)
        )

    def forward(self, x):
        x = self.features(x)
        return self.classifier(x)

x = torch.randn(4, 1, 32, 32)
model = LeNet()
print(model(x).shape)  # batch of 4 -> 10 logits each
Key takeaways
  • Historically important for handwritten digit recognition.
  • Still excellent for teaching shape calculations through CNNs.

5) AlexNet

Classic CNN

AlexNet helped trigger the deep learning revolution in computer vision. Compared with earlier CNNs, it used deeper stacks, ReLU activations, dropout, and large-scale GPU training.

Main formulas
\[ \text{Conv}(11\times 11, s=4) \rightarrow \text{ReLU} \rightarrow \text{Pool} \rightarrow \cdots \rightarrow \text{FC} \]
\[ \mathrm{ReLU}(x)=\max(0,x) \]
Numerical example

For an input of size \(227 imes 227\), an \(11 imes 11\) convolution with stride \(4\) produces \[ \left\lfloor rac{227-11}{4} ight floor + 1 = 55 \] positions along each axis, so each output map is \(55 imes 55\).

Annotated Python code 
# AlexNet is available directly in torchvision
import torch
from torchvision import models

# Create model without downloading pretrained weights
model = models.alexnet(weights=None)

# Fake RGB image batch: 2 images, 3 channels, 224x224
x = torch.randn(2, 3, 224, 224)
logits = model(x)

print("AlexNet output shape:", logits.shape)
Key takeaways
  • Large receptive fields early in the network.
  • Introduced dropout as an important regularizer in CNN pipelines.

6) VGG

Classic CNN

VGG showed that depth alone can be a powerful design principle. It uses repeated small \(3 imes 3\) convolutions instead of large filters, making the architecture conceptually simple and modular.

Main formulas
\[ \text{stack of } 3\times 3 \text{ convolutions} \Rightarrow \text{larger effective receptive field} \]
\[ \text{Two } 3\times 3 \text{ layers} \approx \text{one } 5\times 5 \text{ receptive field, but with extra nonlinearity} \]
Numerical example

A single \(5 imes 5\) convolution has \(25C_ ext{in}C_ ext{out}\) weights. Two \(3 imes 3\) convolutions have \(18C^2\) weights when channel counts match, and they insert an extra nonlinearity between layers.

Annotated Python code 
# VGG-style network from torchvision
import torch
from torchvision import models

model = models.vgg16(weights=None)

# Forward pass through a standard VGG
x = torch.randn(1, 3, 224, 224)
y = model(x)

print("VGG16 output shape:", y.shape)
Key takeaways
  • Simple and strong baseline for feature extraction.
  • Heavy in memory and parameters compared with newer efficient models.

7) ResNet

Residual

ResNet adds identity skip connections so a block learns a residual function instead of an entire transformation from scratch. This makes optimization easier and allows very deep models to train reliably.

Main formulas
\[ \mathbf{y} = \mathcal{F}(\mathbf{x}) + \mathbf{x} \]
\[ \mathbf{y} = \phi(\mathcal{F}(\mathbf{x}) + \mathbf{x}) \]
Numerical example

If a block input is \(\mathbf{x}=[1,-2]\) and the learned residual is \(\mathcal{F}(\mathbf{x})=[0.2,0.5]\), then the block output before activation is \([1.2,-1.5]\). The network only had to learn the correction, not the full mapping.

Annotated Python code 
# Residual block example
import torch
import torch.nn as nn

class ResidualBlock(nn.Module):
    def __init__(self, channels):
        super().__init__()
        self.block = nn.Sequential(
            nn.Conv2d(channels, channels, 3, padding=1),
            nn.ReLU(),
            nn.Conv2d(channels, channels, 3, padding=1)
        )
        self.act = nn.ReLU()

    def forward(self, x):
        # Add the skip path directly to the learned residual
        return self.act(self.block(x) + x)

x = torch.randn(1, 16, 32, 32)
block = ResidualBlock(16)
print(block(x).shape)
Key takeaways
  • Core idea behind many modern image and multimodal backbones.
  • Residual connections are now used well beyond CNNs.

8) DenseNet

Residual / Dense

DenseNet connects each layer to all later layers within a dense block. Earlier features are reused by concatenation, which encourages feature diversity and efficient gradient flow.

Main formulas
\[ \mathbf{x}_\ell = H_\ell([\mathbf{x}_0; \mathbf{x}_1; \ldots; \mathbf{x}_{\ell-1}]) \]
\[ [\cdot ; \cdot] \text{ denotes concatenation along the channel dimension} \]
Numerical example

If layer 0 outputs 16 channels and each new layer adds 8 channels, then after three growth steps the block contains \(16 + 3 imes 8 = 40\) channels. The growth rate controls how quickly width increases.

Annotated Python code 
# Simple DenseNet-style concatenation idea
import torch
import torch.nn as nn

x0 = torch.randn(1, 8, 16, 16)

layer1 = nn.Conv2d(8, 4, kernel_size=3, padding=1)
x1 = torch.relu(layer1(x0))

# Concatenate old and new features
x_cat = torch.cat([x0, x1], dim=1)

layer2 = nn.Conv2d(12, 4, kernel_size=3, padding=1)
x2 = torch.relu(layer2(x_cat))

print("x0 channels :", x0.shape[1])
print("after concat :", x_cat.shape[1])
print("new output   :", x2.shape[1])
Key takeaways
  • Encourages feature reuse instead of repeatedly relearning similar filters.
  • Concatenation can increase memory use.

9) Inception / GoogLeNet

Multi-branch CNN

Inception uses parallel branches with different kernel sizes so the model can capture information at multiple spatial scales in the same layer.

Main formulas
\[ \text{Output} = [\text{branch}_{1\times 1}; \text{branch}_{3\times 3}; \text{branch}_{5\times 5}; \text{pool branch}] \]
\[ 1\times 1 \text{ convolutions reduce channels before expensive branches} \]
Numerical example

Suppose four branches produce 8, 16, 8, and 4 channels. After concatenation, the module output has \(8+16+8+4=36\) channels.

Annotated Python code 
# Tiny Inception-style module
import torch
import torch.nn as nn

class InceptionMini(nn.Module):
    def __init__(self, in_ch):
        super().__init__()
        self.b1 = nn.Conv2d(in_ch, 8, kernel_size=1)
        self.b2 = nn.Conv2d(in_ch, 8, kernel_size=3, padding=1)
        self.b3 = nn.Conv2d(in_ch, 8, kernel_size=5, padding=2)
        self.pool = nn.Sequential(
            nn.MaxPool2d(3, stride=1, padding=1),
            nn.Conv2d(in_ch, 4, kernel_size=1)
        )

    def forward(self, x):
        return torch.cat([self.b1(x), self.b2(x), self.b3(x), self.pool(x)], dim=1)

x = torch.randn(1, 3, 32, 32)
m = InceptionMini(3)
print(m(x).shape)  # 8 + 8 + 8 + 4 channels
Key takeaways
  • Multi-scale pattern extraction in one module.
  • A precursor to many later architectural search ideas.

10) MobileNet

Efficient CNN

MobileNet reduces computation using depthwise separable convolution: a depthwise convolution filters each channel independently, then a pointwise \(1 imes 1\) convolution mixes channels.

Main formulas
\[ \text{Standard conv cost} \approx D_k^2 M N D_f^2 \]
\[ \text{Depthwise separable cost} \approx D_k^2 M D_f^2 + M N D_f^2 \]
Numerical example

With kernel size \(3\), \(M=32\) input channels, \(N=64\) output channels, and feature map size \(D_f=56\), depthwise separable convolution uses far fewer multiply-add operations than a full convolution.

Annotated Python code 
# Depthwise separable convolution in PyTorch
import torch
import torch.nn as nn

depthwise = nn.Conv2d(32, 32, kernel_size=3, padding=1, groups=32)
pointwise = nn.Conv2d(32, 64, kernel_size=1)

x = torch.randn(1, 32, 56, 56)
y = pointwise(torch.relu(depthwise(x)))

print("input :", x.shape)
print("output:", y.shape)
Key takeaways
  • Strong choice for mobile and edge devices.
  • Often paired with width multipliers for further compression.

11) EfficientNet

Efficient CNN

EfficientNet scales network depth, width, and image resolution together rather than scaling only one dimension. This produces better accuracy-efficiency tradeoffs than naive scaling.

Main formulas
\[ d = \alpha^\phi,\quad w = \beta^\phi,\quad r = \gamma^\phi \]
\[ \alpha \beta^2 \gamma^2 \approx 2 \quad \text{under a doubled-compute constraint} \]
Numerical example

If \(\phi=1\), and the chosen coefficients satisfy \(lpha=1.2\), \(eta=1.1\), \(\gamma=1.15\), then the next model is slightly deeper, wider, and uses higher-resolution inputs, instead of scaling only one part.

Annotated Python code 
# EfficientNet from torchvision
import torch
from torchvision import models

model = models.efficientnet_b0(weights=None)
x = torch.randn(1, 3, 224, 224)
y = model(x)

print("EfficientNet-B0 output shape:", y.shape)
Key takeaways
  • Excellent accuracy per FLOP for many image tasks.
  • Compound scaling is the key conceptual contribution.

12) Recurrent Neural Network (RNN)

Sequence

An RNN processes sequences one step at a time, carrying a hidden state forward. It is the simplest recurrent architecture for modeling temporal dependence.

Main formulas
\[ \mathbf{h}_t = \tanh(\mathbf{W}_{xh}\mathbf{x}_t + \mathbf{W}_{hh}\mathbf{h}_{t-1} + \mathbf{b}_h) \]
\[ \mathbf{o}_t = \mathbf{W}_{ho}\mathbf{h}_t + \mathbf{b}_o \]
Numerical example

With scalar inputs and hidden state, let \(x_t=1\), \(h_{t-1}=0.5\), \(W_{xh}=0.8\), \(W_{hh}=0.4\), \(b=0\). Then \(h_t= anh(0.8+0.2)pprox anh(1.0)pprox 0.762\).

Annotated Python code 
# Basic RNN layer
import torch
import torch.nn as nn

# Sequence length 5, batch size 2, input size 3
x = torch.randn(5, 2, 3)

rnn = nn.RNN(input_size=3, hidden_size=4)
y, h_last = rnn(x)

print("all hidden states:", y.shape)   # (seq, batch, hidden)
print("final hidden state:", h_last.shape)
Key takeaways
  • Simple and instructive, but struggles with long-term dependencies.
  • Vanishing and exploding gradients motivated LSTM and GRU.

13) Long Short-Term Memory (LSTM)

Sequence

LSTM adds gates and a cell state so information can be kept, updated, or discarded more selectively than in a vanilla RNN.

Main formulas
\[ \mathbf{f}_t = \sigma(\mathbf{W}_f[\mathbf{h}_{t-1},\mathbf{x}_t]+\mathbf{b}_f) \]
\[ \mathbf{i}_t = \sigma(\mathbf{W}_i[\mathbf{h}_{t-1},\mathbf{x}_t]+\mathbf{b}_i),\quad \tilde{\mathbf{c}}_t=\tanh(\mathbf{W}_c[\mathbf{h}_{t-1},\mathbf{x}_t]+\mathbf{b}_c) \]
\[ \mathbf{c}_t = \mathbf{f}_t \odot \mathbf{c}_{t-1} + \mathbf{i}_t \odot \tilde{\mathbf{c}}_t,\quad \mathbf{h}_t = \mathbf{o}_t \odot \tanh(\mathbf{c}_t) \]
Numerical example

Suppose \(f_t=0.9\), \(i_t=0.2\), \(c_{t-1}=1.0\), and \( ilde{c}_t=0.5\). Then \(c_t = 0.9(1.0) + 0.2(0.5)=1.0\). The forget gate kept most past memory while the input gate added only a small update.

Annotated Python code 
# LSTM forward pass on a toy sequence
import torch
import torch.nn as nn

x = torch.randn(7, 3, 5)  # seq=7, batch=3, features=5

lstm = nn.LSTM(input_size=5, hidden_size=6, num_layers=1)
y, (h_n, c_n) = lstm(x)

print("sequence outputs:", y.shape)
print("final hidden    :", h_n.shape)
print("final cell      :", c_n.shape)
Key takeaways
  • Widely used for time series, biosignals, and text before Transformers became dominant.
  • Cell state is the main mechanism for long-range memory.

14) Gated Recurrent Unit (GRU)

Sequence

GRU is a streamlined alternative to LSTM. It uses fewer gates and no separate cell state, which often makes it faster while remaining effective.

Main formulas
\[ \mathbf{z}_t = \sigma(\mathbf{W}_z[\mathbf{h}_{t-1},\mathbf{x}_t]) \]
\[ \mathbf{r}_t = \sigma(\mathbf{W}_r[\mathbf{h}_{t-1},\mathbf{x}_t]) \]
\[ \tilde{\mathbf{h}}_t = \tanh(\mathbf{W}_h[\mathbf{r}_t \odot \mathbf{h}_{t-1}, \mathbf{x}_t]) \]
\[ \mathbf{h}_t = (1-\mathbf{z}_t)\odot \mathbf{h}_{t-1} + \mathbf{z}_t \odot \tilde{\mathbf{h}}_t \]
Numerical example

If \(z_t=0.7\), \(h_{t-1}=0.2\), and \( ilde{h}_t=0.9\), then \(h_t = 0.3(0.2) + 0.7(0.9) = 0.69\). The update gate controlled how much of the candidate state replaced the old state.

Annotated Python code 
# GRU layer
import torch
import torch.nn as nn

x = torch.randn(10, 4, 8)  # seq=10, batch=4, features=8
gru = nn.GRU(input_size=8, hidden_size=12)

y, h_n = gru(x)
print("outputs:", y.shape)
print("last hidden:", h_n.shape)
Key takeaways
  • Often a practical alternative to LSTM when you want fewer parameters.
  • Still recurrent, so sequence processing is not fully parallel across time.

15) Bidirectional LSTM

Sequence

A bidirectional LSTM reads the sequence in both forward and backward directions, then combines both hidden states. This is helpful when the full sequence is available and both past and future context matter.

Main formulas
\[ \overrightarrow{\mathbf{h}}_t = \mathrm{LSTM}_f(\mathbf{x}_t,\overrightarrow{\mathbf{h}}_{t-1}) \]
\[ \overleftarrow{\mathbf{h}}_t = \mathrm{LSTM}_b(\mathbf{x}_t,\overleftarrow{\mathbf{h}}_{t+1}) \]
\[ \mathbf{h}_t = [\overrightarrow{\mathbf{h}}_t ; \overleftarrow{\mathbf{h}}_t] \]
Numerical example

If the forward hidden state at a token is length 16 and the backward hidden state is also length 16, their concatenation produces a 32-dimensional contextual representation.

Annotated Python code 
# Bidirectional LSTM
import torch
import torch.nn as nn

x = torch.randn(6, 2, 4)  # seq=6, batch=2, features=4

bilstm = nn.LSTM(input_size=4, hidden_size=5, bidirectional=True)
y, (h_n, c_n) = bilstm(x)

print("token representations:", y.shape)  # hidden becomes 2*5
print("final hidden states  :", h_n.shape)
Key takeaways
  • Common in tagging, speech, and biological sequence labeling.
  • Not suitable when only past context is allowed at inference time.

\[ \mathbf{s}_t = f_{dec}(\mathbf{y}_{t-1}, \mathbf{s}_{t-1}, \mathbf{c}) \]
\[ p(y_t \mid y_{
Numerical example

If the encoder compresses a 5-token source sequence into a context vector of length 64, the decoder uses that context plus previously generated tokens to emit the next target token at each time step.

Annotated Python code

import torch
import torch.nn as nn

Key takeaways
  • Historically central in machine translation and summarization.
  • Attention made seq2seq much stronger by removing the fixed-size bottleneck.

17) Additive / Soft Attention

Attention

Attention lets a model focus on the most relevant input positions when producing an output. Instead of forcing all information through a single vector, it computes a weighted combination of encoder states.

Main formulas
\[ e_{t,i} = \mathrm{score}(\mathbf{s}_{t-1}, \mathbf{h}_i) \]
\[ \alpha_{t,i} = \frac{\exp(e_{t,i})}{\sum_j \exp(e_{t,j})} \]
\[ \mathbf{c}_t = \sum_i \alpha_{t,i}\mathbf{h}_i \]
Numerical example

Suppose the alignment scores for three source positions are \([1,2,0]\). Then \[ \mathrm{softmax}([1,2,0]) pprox [0.245, 0.665, 0.090]. \] The second position receives the most attention, so it contributes most to the context vector.

Annotated Python code 
# Manual attention weight computation
import torch

# Three encoder states, each of dimension 4
H = torch.tensor([
    [1.0, 0.0, 1.0, 0.0],
    [0.5, 1.0, 0.5, 1.0],
    [0.0, 1.0, 0.0, 1.0]
])

# Pretend these are the attention scores for one decoder step
scores = torch.tensor([1.0, 2.0, 0.0])

# Turn scores into a probability distribution
alpha = torch.softmax(scores, dim=0)

# Weighted average of encoder states
context = (alpha[:, None] * H).sum(dim=0)

print("attention weights:", alpha)
print("context vector:", context)
Key takeaways
  • Core concept that paved the way for Transformers.
  • The weights are often interpretable as relevance scores, though not always perfectly.

18) Transformer

Attention

The Transformer replaces recurrence with self-attention, allowing each token to directly interact with all others in the sequence. This makes training highly parallel and gives excellent long-range modeling power.

Main formulas
\[ \mathrm{Attention}(\mathbf{Q},\mathbf{K},\mathbf{V}) = \mathrm{softmax}\!\left(\frac{\mathbf{Q}\mathbf{K}^\top}{\sqrt{d_k}}\right)\mathbf{V} \]
\[ \mathrm{MultiHead}(\mathbf{X}) = \mathrm{Concat}(\text{head}_1,\ldots,\text{head}_h)\mathbf{W}^O \]
\[ \mathbf{X}' = \mathrm{LayerNorm}(\mathbf{X} + \mathrm{Attention}(\mathbf{X})) \]
Numerical example

If a query and three keys produce scaled scores \([0.1, 1.2, -0.3]\), their softmax weights may be roughly \([0.21, 0.64, 0.15]\), so the output is mostly the value vector from the second token.

Annotated Python code 
# Transformer encoder block from PyTorch
import torch
import torch.nn as nn

layer = nn.TransformerEncoderLayer(
    d_model=32,      # embedding dimension
    nhead=4,         # four attention heads
    dim_feedforward=64,
    batch_first=True
)

# Batch of 2 sequences, each with 6 tokens, embedding size 32
x = torch.randn(2, 6, 32)
y = layer(x)

print("input shape :", x.shape)
print("output shape:", y.shape)
Key takeaways
  • Dominant architecture for language and widely used in vision, biology, audio, and multimodal learning.
  • Needs positional information because self-attention alone is permutation-invariant.

19) BERT (Bidirectional Encoder Representations from Transformers)

Language

BERT is a Transformer encoder trained to predict masked tokens from both left and right context. It produces contextual embeddings that are highly effective for downstream classification and token-level tasks.

Main formulas
\[ p(x_m \mid x_{\setminus m}) = \mathrm{softmax}(\mathbf{W}\mathbf{h}_m) \]
\[ \text{BERT uses bidirectional self-attention over the full input sequence} \]
Numerical example

In the sentence “[CLS] the cell is [MASK] rapidly”, BERT uses both “the cell is” and “rapidly” to predict a word such as “dividing”. This differs from left-to-right generation because the masked position can see both sides.

Annotated Python code 
# Tiny BERT-like encoder idea using embeddings + Transformer encoder
import torch
import torch.nn as nn

vocab_size = 100
d_model = 32

embedding = nn.Embedding(vocab_size, d_model)
encoder = nn.TransformerEncoder(
    nn.TransformerEncoderLayer(d_model=d_model, nhead=4, batch_first=True),
    num_layers=2
)
mlm_head = nn.Linear(d_model, vocab_size)

# Batch of token IDs
tokens = torch.randint(0, vocab_size, (2, 8))
hidden = encoder(embedding(tokens))
logits = mlm_head(hidden)

print("token logits shape:", logits.shape)  # (batch, seq, vocab)
Key takeaways
  • Usually fine-tuned for classification, tagging, ranking, and embedding extraction.
  • Encoder-only: good for understanding tasks, not autoregressive generation.

20) GPT-style Decoder Transformer

Language

GPT is a decoder-only Transformer trained to predict the next token. The causal mask ensures each position can only use earlier positions, making the model suitable for text generation.

Main formulas
\[ p(x_1,\ldots,x_T) = \prod_{t=1}^{T} p(x_t \mid x_{
\[ \text{causal mask blocks attention to future positions} \]
Numerical example

If the context is “deep learning models are”, GPT estimates a probability distribution over the next token, for example giving higher mass to “powerful”, “useful”, or “widely” than unrelated words.

Annotated Python code 
# Causal Transformer decoder-style forward pass
import torch
import torch.nn as nn

d_model = 32
vocab_size = 120

embedding = nn.Embedding(vocab_size, d_model)
decoder_layer = nn.TransformerEncoderLayer(d_model=d_model, nhead=4, batch_first=True)
decoder = nn.TransformerEncoder(decoder_layer, num_layers=2)
lm_head = nn.Linear(d_model, vocab_size)

tokens = torch.randint(0, vocab_size, (2, 6))
x = embedding(tokens)

# Build a causal mask so future tokens cannot be used
seq_len = tokens.size(1)
mask = torch.triu(torch.ones(seq_len, seq_len) * float("-inf"), diagonal=1)

hidden = decoder(x, mask=mask)
logits = lm_head(hidden)

print("language model logits:", logits.shape)
Key takeaways
  • Decoder-only Transformers dominate modern generative language modeling.
  • Training objective is conceptually simple: next-token prediction.

21) Autoencoder

Representation Learning

An autoencoder learns to compress an input into a latent representation and reconstruct it. The bottleneck forces the model to capture the most informative structure in the data.

Main formulas
\[ \mathbf{z} = f_\theta(\mathbf{x}) \]
\[ \hat{\mathbf{x}} = g_\phi(\mathbf{z}) \]
\[ \mathcal{L} = \|\mathbf{x}-\hat{\mathbf{x}}\|_2^2 \]
Numerical example

Suppose a 10-dimensional input is compressed into a latent vector of length 2. If reconstruction remains accurate, those 2 latent dimensions have captured a compact summary of the original pattern.

Annotated Python code 
# Small fully connected autoencoder
import torch
import torch.nn as nn

class Autoencoder(nn.Module):
    def __init__(self):
        super().__init__()
        self.encoder = nn.Sequential(
            nn.Linear(10, 4),
            nn.ReLU(),
            nn.Linear(4, 2)   # bottleneck
        )
        self.decoder = nn.Sequential(
            nn.Linear(2, 4),
            nn.ReLU(),
            nn.Linear(4, 10)
        )

    def forward(self, x):
        z = self.encoder(x)
        x_hat = self.decoder(z)
        return x_hat, z

x = torch.randn(3, 10)
model = Autoencoder()
x_hat, z = model(x)

print("latent shape:", z.shape)
print("reconstruction shape:", x_hat.shape)
Key takeaways
  • Useful for compression, denoising, anomaly detection, and latent-space exploration.
  • Too much capacity can let the model memorize instead of learn structure.

22) Denoising Autoencoder

Representation Learning

A denoising autoencoder learns to recover the clean input from a corrupted version. This encourages the latent representation to focus on stable structure rather than raw noise.

Main formulas
\[ \tilde{\mathbf{x}} = \mathbf{x} + \epsilon \]
\[ \hat{\mathbf{x}} = g_\phi(f_\theta(\tilde{\mathbf{x}})) \]
\[ \mathcal{L} = \|\mathbf{x} - \hat{\mathbf{x}}\|_2^2 \]
Numerical example

If the clean vector is \([1,0,1,0]\) and Gaussian noise perturbs it to \([0.8,0.1,1.2,-0.2]\), the model is trained to reconstruct the clean target, not the noisy version.

Annotated Python code 
# Denoising autoencoder idea
import torch
import torch.nn as nn

encoder = nn.Sequential(nn.Linear(6, 3), nn.ReLU())
decoder = nn.Linear(3, 6)

x_clean = torch.randn(4, 6)

# Add Gaussian noise to inputs
noise = 0.2 * torch.randn_like(x_clean)
x_noisy = x_clean + noise

z = encoder(x_noisy)
x_hat = decoder(z)

loss = torch.mean((x_clean - x_hat) ** 2)
print("denoising loss:", loss.item())
Key takeaways
  • Improves robustness and often learns better latent representations than a plain autoencoder.
  • Noise type can be Gaussian, masking, salt-and-pepper, and more.

23) Variational Autoencoder (VAE)

Generative

A VAE is a probabilistic autoencoder. Instead of mapping each input to a fixed code, it maps to a distribution in latent space and regularizes that latent space toward a simple prior.

Main formulas
\[ q_\phi(\mathbf{z}\mid \mathbf{x}) = \mathcal{N}(\boldsymbol{\mu}(\mathbf{x}), \mathrm{diag}(\boldsymbol{\sigma}^2(\mathbf{x}))) \]
\[ \mathbf{z} = \boldsymbol{\mu} + \boldsymbol{\sigma}\odot \boldsymbol{\epsilon}, \quad \boldsymbol{\epsilon}\sim \mathcal{N}(0,\mathbf{I}) \]
\[ \mathcal{L} = \underbrace{\|\mathbf{x}-\hat{\mathbf{x}}\|^2}_{\text{reconstruction}} + \underbrace{D_{KL}(q_\phi(\mathbf{z}\mid\mathbf{x}) \,\|\, p(\mathbf{z}))}_{\text{regularization}} \]
Numerical example

For \(\mu=[0.2,-0.1]\) and \(\log \sigma^2=[-0.2,0.1]\), the KL term for each dimension is \[ - frac12(1 + \log\sigma^2 - \mu^2 - \sigma^2). \] This penalizes latent distributions that drift too far from the standard normal prior.

Annotated Python code 
# VAE encoder pieces: mu, logvar, reparameterization
import torch
import torch.nn as nn

encoder = nn.Linear(8, 6)
to_mu = nn.Linear(6, 2)
to_logvar = nn.Linear(6, 2)
decoder = nn.Linear(2, 8)

x = torch.randn(5, 8)
h = torch.relu(encoder(x))
mu = to_mu(h)
logvar = to_logvar(h)

# Reparameterization trick
std = torch.exp(0.5 * logvar)
eps = torch.randn_like(std)
z = mu + std * eps

x_hat = decoder(z)

# Mean KL divergence to N(0, I)
kl = -0.5 * torch.mean(1 + logvar - mu.pow(2) - logvar.exp())
print("recon shape:", x_hat.shape)
print("KL term:", kl.item())
Key takeaways
  • Useful for generative modeling and smooth latent interpolation.
  • Tends to generate blurrier images than strong GANs or diffusion models.

24) Generative Adversarial Network (GAN)

Generative

A GAN trains two networks in opposition: a generator produces fake samples, and a discriminator tries to distinguish real from fake. The adversarial game pushes the generator toward realistic outputs.

Main formulas
\[ \min_G \max_D \; \mathbb{E}_{x\sim p_{data}}[\log D(x)] + \mathbb{E}_{z\sim p(z)}[\log(1-D(G(z)))] \]
\[ G: \mathbf{z}\mapsto \hat{\mathbf{x}}, \quad D: \mathbf{x}\mapsto [0,1] \]
Numerical example

If the discriminator gives \(D(x_{real})=0.9\) and \(D(G(z))=0.2\), then it currently distinguishes real and fake well. Training updates the generator to increase \(D(G(z))\) and fool the discriminator.

Annotated Python code 
# Very small GAN components
import torch
import torch.nn as nn

generator = nn.Sequential(
    nn.Linear(5, 16),
    nn.ReLU(),
    nn.Linear(16, 8)
)

discriminator = nn.Sequential(
    nn.Linear(8, 16),
    nn.ReLU(),
    nn.Linear(16, 1),
    nn.Sigmoid()
)

z = torch.randn(4, 5)      # latent noise
fake = generator(z)        # generated samples
scores = discriminator(fake)

print("fake samples shape:", fake.shape)
print("discriminator scores:", scores.squeeze())
Key takeaways
  • Excellent for sharp synthetic samples, but training can be unstable.
  • Mode collapse is a classic GAN failure mode.

25) Conditional GAN (cGAN)

Generative

A conditional GAN guides generation using an external condition such as a class label, style tag, or auxiliary measurement. Both generator and discriminator receive the condition so outputs can be targeted.

Main formulas
\[ G(\mathbf{z}, \mathbf{y}) \rightarrow \hat{\mathbf{x}} \]
\[ D(\mathbf{x}, \mathbf{y}) \rightarrow [0,1] \]
Numerical example

If the condition is the class label “cat”, the generator is trained to produce cat-like samples rather than arbitrary images. Changing the condition to “dog” should shift the generated distribution accordingly.

Annotated Python code 
# Conditional GAN idea: concatenate latent vector with one-hot label
import torch
import torch.nn as nn

n_classes = 3
z_dim = 5

generator = nn.Sequential(
    nn.Linear(z_dim + n_classes, 16),
    nn.ReLU(),
    nn.Linear(16, 8)
)

# One latent sample and one class condition
z = torch.randn(1, z_dim)
label = torch.tensor([[0., 1., 0.]])  # class 1 in one-hot form

g_input = torch.cat([z, label], dim=1)
fake = generator(g_input)

print("conditional fake sample:", fake)
Key takeaways
  • Used in class-conditional image synthesis, super-resolution, and paired translation setups.
  • Conditioning adds controllability to adversarial generation.

26) U-Net

Segmentation

U-Net is an encoder–decoder architecture for dense prediction such as image segmentation. Its skip connections pass high-resolution spatial detail from the encoder to the decoder, which is crucial for precise localization.

Main formulas
\[ \text{Encoder features} \rightarrow \text{bottleneck} \rightarrow \text{Decoder features} \]
\[ \text{Decoder input at level } \ell = [\text{upsampled feature}_\ell ; \text{encoder feature}_\ell] \]
Numerical example

If the encoder creates a \(64 imes 64\) feature map with 32 channels and the decoder upsamples a matching \(64 imes 64\) map with 32 channels, their concatenation yields 64 channels before the next convolution block.

Annotated Python code 
# Very small U-Net-like skeleton
import torch
import torch.nn as nn

enc1 = nn.Conv2d(1, 8, 3, padding=1)
pool = nn.MaxPool2d(2)
enc2 = nn.Conv2d(8, 16, 3, padding=1)

up = nn.ConvTranspose2d(16, 8, kernel_size=2, stride=2)
dec = nn.Conv2d(16, 8, 3, padding=1)  # receives skip + upsampled
out = nn.Conv2d(8, 1, 1)

x = torch.randn(1, 1, 64, 64)
s1 = torch.relu(enc1(x))
b = torch.relu(enc2(pool(s1)))
u = up(b)

# Skip connection: concatenate encoder and decoder features
cat = torch.cat([u, s1], dim=1)
y = out(torch.relu(dec(cat)))

print("segmentation output shape:", y.shape)
Key takeaways
  • A standard baseline for biomedical and remote-sensing segmentation.
  • Skip connections restore fine structure lost during downsampling.

27) Siamese Network

Metric Learning

A Siamese network learns an embedding space where similar inputs are close and dissimilar inputs are far apart. The two branches share weights, so both inputs are transformed by the same encoder.

Main formulas
\[ \mathbf{e}_1 = f_\theta(\mathbf{x}_1), \quad \mathbf{e}_2 = f_\theta(\mathbf{x}_2) \]
\[ d = \|\mathbf{e}_1 - \mathbf{e}_2\|_2 \]
\[ \mathcal{L} = y d^2 + (1-y)\max(0, m-d)^2 \]
Numerical example

If two positive examples produce embeddings with distance \(d=0.3\), the contrastive loss for \(y=1\) is small. If a negative pair has \(d=0.4\) but margin \(m=1\), it is penalized because the pair is still too close.

Annotated Python code 
# Siamese embedding distance
import torch
import torch.nn as nn

encoder = nn.Sequential(
    nn.Linear(6, 4),
    nn.ReLU(),
    nn.Linear(4, 2)
)

x1 = torch.randn(1, 6)
x2 = torch.randn(1, 6)

e1 = encoder(x1)
e2 = encoder(x2)

distance = torch.norm(e1 - e2, p=2)
print("embedding 1:", e1)
print("embedding 2:", e2)
print("Euclidean distance:", distance.item())
Key takeaways
  • Useful for one-shot learning, duplicate detection, and similarity search.
  • Triplet loss is a close relative of contrastive loss.

28) Graph Convolutional Network (GCN)

Graph

A GCN propagates information over graph edges so each node updates its representation using its neighbors. This is useful when relationships are not arranged on a regular grid.

Main formulas
\[ \mathbf{H}^{(\ell+1)} = \sigma(\hat{\mathbf{A}}\mathbf{H}^{(\ell)}\mathbf{W}^{(\ell)}) \]
\[ \hat{\mathbf{A}} = \tilde{\mathbf{D}}^{-1/2}\tilde{\mathbf{A}}\tilde{\mathbf{D}}^{-1/2}, \quad \tilde{\mathbf{A}} = \mathbf{A} + \mathbf{I} \]
Numerical example

If node 2 is connected to nodes 1 and 3, then its next representation is a normalized mixture of its own features and those of its neighbors. The normalization prevents high-degree nodes from dominating the aggregation.

Annotated Python code 
# One manual GCN layer using plain PyTorch
import torch

# Adjacency for a 3-node chain plus self-loops
A = torch.tensor([
    [1., 1., 0.],
    [1., 1., 1.],
    [0., 1., 1.]
])

# Node feature matrix: 3 nodes x 2 features
H = torch.tensor([
    [1., 0.],
    [0., 1.],
    [1., 1.]
])

# Symmetric normalization
D = torch.diag(torch.sum(A, dim=1) ** -0.5)
A_hat = D @ A @ D

# Learnable weights for a graph layer
W = torch.randn(2, 4)

# Message passing + linear projection + nonlinearity
H_next = torch.relu(A_hat @ H @ W)
print(H_next)
Key takeaways
  • Common in node classification, molecular graphs, and biological interaction networks.
  • Message passing is the core organizing idea in graph neural networks.

29) Graph Attention Network (GAT)

Graph

GAT improves graph aggregation by learning attention weights over neighbors. Instead of treating all adjacent nodes equally, it decides which neighbors are most informative for the current node.

Main formulas
\[ e_{ij} = \mathrm{LeakyReLU}\!\left(\mathbf{a}^\top[\mathbf{W}\mathbf{h}_i \, \| \, \mathbf{W}\mathbf{h}_j]\right) \]
\[ \alpha_{ij} = \mathrm{softmax}_{j\in \mathcal{N}(i)}(e_{ij}) \]
\[ \mathbf{h}'_i = \sigma\!\left(\sum_{j\in\mathcal{N}(i)} \alpha_{ij}\mathbf{W}\mathbf{h}_j\right) \]
Numerical example

If node \(i\) has three neighbors with unnormalized scores \([0.2, 1.0, -0.1]\), their softmax weights might be \([0.25, 0.55, 0.20]\), so the second neighbor contributes most to the update.

Annotated Python code 
# Tiny hand-computed attention weights for one graph node
import torch

scores = torch.tensor([0.2, 1.0, -0.1])   # attention scores to three neighbors
alpha = torch.softmax(scores, dim=0)

neighbor_messages = torch.tensor([
    [1.0, 0.0],
    [0.5, 1.0],
    [0.0, 1.0]
])

updated = (alpha[:, None] * neighbor_messages).sum(dim=0)

print("attention weights:", alpha)
print("updated node representation:", updated)
Key takeaways
  • Better than plain averaging when some neighbors are much more relevant than others.
  • Can be more expressive but also a bit more expensive than a simple GCN.

30) Diffusion Model

Generative

Diffusion models learn to reverse a gradual noising process. During training, noise is added to data at many levels, and the network learns to predict or remove that noise.

Main formulas
\[ q(\mathbf{x}_t \mid \mathbf{x}_{t-1}) = \mathcal{N}(\sqrt{1-\beta_t}\mathbf{x}_{t-1}, \beta_t \mathbf{I}) \]
\[ \mathbf{x}_t = \sqrt{\bar{\alpha}_t}\mathbf{x}_0 + \sqrt{1-\bar{\alpha}_t}\boldsymbol{\epsilon} \]
\[ \mathcal{L} = \mathbb{E}_{t,\mathbf{x}_0,\boldsymbol{\epsilon}} \big[\|\boldsymbol{\epsilon} - \boldsymbol{\epsilon}_\theta(\mathbf{x}_t,t)\|_2^2\big] \]
Numerical example

If \(ar{lpha}_t = 0.81\), then \(\sqrt{ar{lpha}_t}=0.9\) and \(\sqrt{1-ar{lpha}_t}pprox 0.436\). So at step \(t\), the noisy sample is a weighted mixture of 90% signal and 43.6% random noise amplitude.

Annotated Python code 
# One forward noising step for diffusion
import torch

x0 = torch.tensor([[1.0, -1.0, 0.5]])   # clean sample
alpha_bar_t = 0.81
eps = torch.randn_like(x0)              # Gaussian noise

# Closed-form noisy sample at time t
xt = (alpha_bar_t ** 0.5) * x0 + ((1 - alpha_bar_t) ** 0.5) * eps

print("clean sample :", x0)
print("noise        :", eps)
print("noisy sample :", xt)
Key takeaways
  • Now a leading approach for high-quality image, audio, and multimodal generation.
  • Training is stable compared with GANs, though sampling can be slower.
