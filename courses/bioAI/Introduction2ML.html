1) Linear Regression

Supervised · Regression

Predicts a continuous outcome by fitting a straight-line or linear hyperplane relationship between features and target.

Core idea

Predicts a continuous outcome by fitting a straight-line or linear hyperplane relationship between features and target.

When it is useful

Predicting continuous outcomes

Formula
\[ \hat{y} = X\beta,\qquad \hat{\beta} = (X^\top X)^{-1}X^\top y \]
Explain the formula
  • \(X\) is the design matrix containing the input features.
  • \(\beta\) contains the coefficients (slopes) and often an intercept term after augmentation.
  • OLS chooses \(\beta\) to minimize the residual sum of squares: \(\sum_i (y_i-\hat y_i)^2\).
Numerical example
Suppose three points are ​(1,2), (2,3), (3,5). The fitted line is approximately \(\hat y = 0.33 + 1.5x\). At \(x=4\), the model predicts about \(6.33\).

2) Ridge Regression

Supervised · Regularized Regression

Adds an L2 penalty to linear regression to shrink coefficients and reduce variance when predictors are noisy or correlated.

Core idea

Adds an L2 penalty to linear regression to shrink coefficients and reduce variance when predictors are noisy or correlated.

When it is useful

Controlling overfitting and unstable coefficients

Formula
\[ \hat{\beta}=\arg\min_{\beta}\left\{\|y-X\beta\|_2^2+\lambda\|\beta\|_2^2\right\}, \qquad \hat{\beta}=(X^\top X+\lambda I)^{-1}X^\top y \]
Explain the formula
  • \(\lambda \ge 0\) controls how strongly coefficients are shrunk toward zero.
  • As \(\lambda\) increases, coefficients get smaller and the model usually becomes more stable.
  • Unlike lasso, ridge usually keeps all predictors but with reduced magnitude.
Numerical example
If two predictors carry nearly duplicate information, OLS can give unstable slopes. Ridge spreads weight more evenly. With \(\lambda=1\), coefficients are shrunk relative to OLS.
Annotated Python code

3) Lasso Regression

Supervised · Sparse Regression

Uses an L1 penalty so some coefficients become exactly zero, making it useful for feature selection.

Core idea

Uses an L1 penalty so some coefficients become exactly zero, making it useful for feature selection.

When it is useful

Selecting a smaller set of predictors

Formula
\[ \hat{\beta}=\arg\min_{\beta}\left\{\|y-X\beta\|_2^2+\lambda\|\beta\|_1\right\} \]
Explain the formula
  • \(\|\beta\|_1=\sum_j |\beta_j|\) encourages sparsity.
  • When \(\lambda\) is large enough, irrelevant features are driven to exactly zero.
  • This is helpful when you have many predictors and want a compact model.
Numerical example
Imagine only the first feature truly matters. Lasso can keep the first coefficient nonzero and shrink the others to zero, effectively selecting a smaller set of variables.

4) Elastic Net

Supervised · Regularized Regression

Combines ridge and lasso penalties, giving both shrinkage and sparse feature selection.

Core idea

Combines ridge and lasso penalties, giving both shrinkage and sparse feature selection.

When it is useful

Controlling overfitting and unstable coefficients

Formula
\[ \hat{\beta}=\arg\min_{\beta}\left\{ \|y-X\beta\|_2^2+ \lambda\left(\alpha\|\beta\|_1+(1-\alpha)\|\beta\|_2^2\right) \right\} \]
Explain the formula
  • \(\alpha=1\) gives lasso-like behavior and \(\alpha=0\) gives ridge-like behavior.
  • It is often useful when predictors are correlated and you still want sparsity.
  • The mix of L1 and L2 usually makes optimization more stable than pure lasso.
Numerical example
If features 1 and 2 are highly correlated and both informative, elastic net can keep both with moderate shrinkage instead of keeping only one arbitrarily.
Annotated Python code

5) Logistic Regression

Supervised · Classification

Models class probability for binary outcomes using the logistic sigmoid.

Core idea

Models class probability for binary outcomes using the logistic sigmoid.

When it is useful

Binary or multiclass label prediction

Formula
\[ p(y=1\mid x)=\sigma(w^\top x+b)=\frac{1}{1+e^{-(w^\top x+b)}} \] \[ \mathcal{L}=-\sum_i \left[y_i\log p_i + (1-y_i)\log(1-p_i)\right] \]
Explain the formula
  • The first formula maps any real-valued score to a probability between 0 and 1.
  • The second formula is the binary cross-entropy loss used for estimation.
  • A threshold such as 0.5 turns probabilities into class labels.
Numerical example
If \(w=2\), \(b=-3\), and \(x=2\), then the score is \(1\) and the probability is \(\sigma(1)\approx 0.731\), so class 1 is more likely.
Annotated Python code

6) k-Nearest Neighbors (kNN)

Supervised · Instance-Based

Predicts a label or value by looking at the nearest training examples under a distance metric such as Euclidean distance.

Core idea

Predicts a label or value by looking at the nearest training examples under a distance metric such as Euclidean distance.

When it is useful

Simple nonparametric local prediction

Formula
\[ d(x,x_i)=\sqrt{\sum_{j=1}^p (x_j-x_{ij})^2} \] \[ \hat{y}=\mathrm{mode}\{y_i: x_i \in \mathcal{N}_k(x)\} \]
Explain the formula
  • The first formula is Euclidean distance between a query point and a training point.
  • \(\mathcal{N}_k(x)\) is the set of the \(k\) nearest neighbors of \(x\).
  • For regression, the prediction is usually the average of neighbors instead of a majority vote.
Numerical example
If the three nearest neighbors to a new point have labels [1, 1, 0], then with \(k=3\) the predicted class is 1.
Annotated Python code

7) Gaussian Naive Bayes

Supervised · Probabilistic Classification

Uses Bayes' rule with a conditional independence assumption and Gaussian feature likelihoods.

Core idea

Uses Bayes' rule with a conditional independence assumption and Gaussian feature likelihoods.

When it is useful

Fast probabilistic classification with simple assumptions

Formula
\[ p(y=k\mid x)\propto p(y=k)\prod_{j=1}^p p(x_j\mid y=k) \] \[ x_j\mid y=k \sim \mathcal{N}(\mu_{kj}, \sigma_{kj}^2) \]
Explain the formula
  • The model multiplies class prior by feature-wise likelihoods.
  • 'Naive' means features are treated as conditionally independent given the class.
  • Despite the strong assumption, it can work surprisingly well, especially on high-dimensional data.
Numerical example
If class A has prior 0.6 and feature likelihood 0.2 while class B has prior 0.4 and likelihood 0.1, the unnormalized posteriors are 0.12 and 0.04, so class A wins.
Annotated Python code

8) Linear Discriminant Analysis (LDA)

Supervised · Generative Classification

Assumes each class is Gaussian with its own mean but a shared covariance matrix, producing linear decision boundaries.

Core idea

Assumes each class is Gaussian with its own mean but a shared covariance matrix, producing linear decision boundaries.

When it is useful

Class separation with Gaussian assumptions

Formula
\[ \delta_k(x)=x^\top \Sigma^{-1}\mu_k - \frac{1}{2}\mu_k^\top\Sigma^{-1}\mu_k + \log \pi_k \]
Explain the formula
  • \(\mu_k\) is the class mean vector and \(\pi_k\) is the class prior.
  • \(\Sigma\) is the covariance matrix shared across classes.
  • The class with the largest discriminant score \(\delta_k(x)\) is predicted.
Numerical example
If two classes have different centers but similar spread, the shared-covariance assumption is reasonable and the boundary is linear.
Annotated Python code

9) Quadratic Discriminant Analysis (QDA)

Supervised · Generative Classification

Like LDA, but each class gets its own covariance matrix, allowing curved or quadratic decision boundaries.

Core idea

Like LDA, but each class gets its own covariance matrix, allowing curved or quadratic decision boundaries.

When it is useful

Class separation with Gaussian assumptions

Formula
\[ \delta_k(x)= -\frac{1}{2}\log |\Sigma_k| -\frac{1}{2}(x-\mu_k)^\top\Sigma_k^{-1}(x-\mu_k) +\log \pi_k \]
Explain the formula
  • \(\Sigma_k\) differs by class, so the geometry can differ class by class.
  • This extra flexibility can capture heteroscedastic classes better than LDA.
  • It needs more data because more covariance parameters must be estimated.
Numerical example
If one class is tightly clustered and another is wide and elongated, QDA can model that difference while LDA cannot.

10) Decision Tree

Supervised · Tree-Based

Recursively splits the feature space into regions that are as pure as possible with respect to the target.

Core idea

Recursively splits the feature space into regions that are as pure as possible with respect to the target.

When it is useful

Interpretable rule-based prediction

Formula
\[ Gini(t)=1-\sum_{k=1}^K p(k\mid t)^2 \] \[ \Delta I = I(\text{parent}) - \sum_{c\in \{\text{left,right}\}} \frac{n_c}{n}I(c) \]
Explain the formula
  • The first formula is Gini impurity for a node \(t\).
  • The second formula measures impurity reduction from a split.
  • Trees are easy to interpret but can overfit if grown too deep.
Numerical example
If a node has class proportions [0.5, 0.5], Gini is 0.5. A split that creates pure children reduces impurity strongly and is preferred.

11) Random Forest

Supervised · Ensemble Trees

Builds many decision trees on bootstrap samples and random feature subsets, then averages or votes across trees.

Core idea

Builds many decision trees on bootstrap samples and random feature subsets, then averages or votes across trees.

When it is useful

Strong general-purpose tabular modeling

Formula
\[ \hat{f}(x)=\frac{1}{B}\sum_{b=1}^B T_b(x) \]
Explain the formula
  • \(T_b(x)\) is the prediction from tree \(b\).
  • Bootstrap sampling and random feature selection decorrelate the trees.
  • The ensemble usually reduces variance compared with a single deep tree.
Numerical example
If five trees vote [1, 1, 0, 1, 0], the final class is 1 by majority vote.

12) Extra Trees (Extremely Randomized Trees)

Supervised · Ensemble Trees

Similar to random forests, but splits are chosen more randomly, which can further reduce variance and training cost.

Core idea

Similar to random forests, but splits are chosen more randomly, which can further reduce variance and training cost.

When it is useful

Strong general-purpose tabular modeling

Formula
\[ \hat{f}(x)=\frac{1}{B}\sum_{b=1}^B T_b^{\text{random-split}}(x) \]
Explain the formula
  • Extra Trees typically use the whole dataset or large subsets instead of bootstrap by default.
  • Candidate split thresholds are more random than in random forests.
  • The extra randomness can improve generalization in some problems.
Numerical example
Instead of searching exhaustively for the best split threshold, the algorithm samples random thresholds and keeps the best among them.

13) AdaBoost

Supervised · Boosting

Builds a strong classifier by sequentially adding weak learners that focus on previously misclassified examples.

Core idea

Builds a strong classifier by sequentially adding weak learners that focus on previously misclassified examples.

When it is useful

High-performance additive ensembles on tabular data

Formula
\[ F(x)=\sum_{t=1}^T \alpha_t h_t(x),\qquad \alpha_t=\frac{1}{2}\log\left(\frac{1-\varepsilon_t}{\varepsilon_t}\right) \]
Explain the formula
  • \(h_t(x)\) is the weak learner at round \(t\), often a shallow tree or stump.
  • \(\varepsilon_t\) is the weighted training error of that learner.
  • Misclassified points get up-weighted so the next learner pays more attention to them.
Numerical example
If a stump has weighted error 0.2, then \(\alpha = 0.5\log(0.8/0.2) \approx 0.693\), so it gets substantial weight in the final vote.
Annotated Python code

14) Gradient Boosting

Supervised · Boosting

Adds weak learners sequentially, each one fitting the residual errors or negative gradient of the loss from the current model.

Core idea

Adds weak learners sequentially, each one fitting the residual errors or negative gradient of the loss from the current model.

When it is useful

High-performance additive ensembles on tabular data

Formula
\[ F_m(x)=F_{m-1}(x)+\nu \gamma_m h_m(x) \]
Explain the formula
  • \(F_{m-1}(x)\) is the current ensemble prediction.
  • \(h_m(x)\) is the new weak learner fit to residual-like targets.
  • \(\nu\) is the learning rate controlling how much each learner contributes.
Numerical example
If the current prediction is 6 but the true target is 8, the residual is 2. The next tree tries to correct errors like that.
Annotated Python code

15) Support Vector Machine (Linear SVM)

Supervised · Margin-Based

Finds a separating hyperplane with maximum margin between classes while allowing some violations through slack variables.

Core idea

Finds a separating hyperplane with maximum margin between classes while allowing some violations through slack variables.

When it is useful

Maximum-margin classification

Formula
\[ \min_{w,b,\xi}\ \frac{1}{2}\|w\|^2 + C\sum_{i=1}^n \xi_i \] \[ y_i(w^\top x_i+b)\ge 1-\xi_i,\qquad \xi_i \ge 0 \]
Explain the formula
  • The first term prefers a large margin because smaller \(\|w\|\) means wider margin.
  • The second term penalizes misclassified or margin-violating points through \(\xi_i\).
  • \(C\) controls the tradeoff between margin width and classification errors.
Numerical example
If two classes are almost linearly separable, the optimal hyperplane is the one that leaves the largest gap between the closest points of the two classes.
Annotated Python code

16) Support Vector Machine (RBF Kernel)

Supervised · Kernel Method

Uses a kernel to create nonlinear decision boundaries without explicitly mapping data to a high-dimensional feature space.

Core idea

Uses a kernel to create nonlinear decision boundaries without explicitly mapping data to a high-dimensional feature space.

When it is useful

Flexible nonlinear classification

Formula
\[ K(x,x')=\exp\left(-\gamma \|x-x'\|^2\right) \]
Explain the formula
  • If two points are close, the kernel value is near 1; if far apart, it approaches 0.
  • The RBF kernel lets SVM capture curved boundaries.
  • \(\gamma\) controls how local the influence of each support vector is.
Numerical example
For two points separated by distance 1 and \(\gamma=0.5\), the kernel similarity is \(e^{-0.5}\approx 0.607\).
Annotated Python code

17) Perceptron

Supervised · Linear Classifier

A simple linear classifier that updates weights only when a training point is misclassified.

Core idea

A simple linear classifier that updates weights only when a training point is misclassified.

When it is useful

Fast online linear learning

Formula
\[ \hat{y}=\mathrm{sign}(w^\top x+b) \] \[ w \leftarrow w + \eta y_i x_i,\qquad b \leftarrow b + \eta y_i \]
Explain the formula
  • If a point is correctly classified, no update is made.
  • If a point is misclassified, the weights are nudged toward the correct class.
  • The perceptron converges when data are linearly separable.
Numerical example
If \(w=(0,0)\), \(\eta=1\), and a positive point \(x=(2,1)\) is misclassified, then the update becomes \(w=(2,1)\).
Annotated Python code

18) Multilayer Perceptron (MLP)

Supervised · Neural Network

Stacks multiple fully connected layers with nonlinear activation functions, enabling the model to learn nonlinear patterns.

Core idea

Stacks multiple fully connected layers with nonlinear activation functions, enabling the model to learn nonlinear patterns.

When it is useful

Learning nonlinear feature interactions

Formula
\[ a^{(l)} = \phi\left(W^{(l)}a^{(l-1)} + b^{(l)}\right) \]
Explain the formula
  • \(a^{(l-1)}\) is the input to layer \(l\), \(W^{(l)}\) are weights, and \(b^{(l)}\) are biases.
  • \(\phi\) is a nonlinear activation such as ReLU or tanh.
  • Backpropagation computes gradients of the loss with respect to all parameters.
Numerical example
A 2→3→1 network can map two inputs into a hidden representation of three learned features before producing a single output.
Annotated Python code

19) Convolutional Neural Network (CNN)

Deep Learning · Structured Inputs

Learns local patterns with convolutional filters, making it powerful for images, signals, and other spatially structured data.

Core idea

Learns local patterns with convolutional filters, making it powerful for images, signals, and other spatially structured data.

When it is useful

Images, grids, motifs, local patterns

Formula
\[ z_{i,j}^{(k)} = \sum_{u}\sum_{v} K_{u,v}^{(k)} X_{i+u,j+v} + b_k \]
Explain the formula
  • The filter \(K^{(k)}\) slides across the input and reuses the same weights everywhere.
  • This creates translation-aware feature detectors such as edge or motif detectors.
  • Pooling layers reduce spatial size and can improve robustness.
Numerical example
A 3×3 edge detector kernel gives high output wherever local pixel patterns look like an edge.
Annotated Python code

20) Recurrent Neural Network / LSTM

Deep Learning · Sequences

Processes sequential inputs one step at a time, carrying information forward through a hidden state. LSTMs use gates to better preserve long-range information.

Core idea

Processes sequential inputs one step at a time, carrying information forward through a hidden state. LSTMs use gates to better preserve long-range information.

When it is useful

Ordered series, language, temporal data, biological sequences

Formula
\[ h_t = f(W_{xh}x_t + W_{hh}h_{t-1} + b) \] \[ c_t = f_t \odot c_{t-1} + i_t \odot \tilde{c}_t \]
Explain the formula
  • The first equation is the basic RNN hidden-state update.
  • The second equation is the LSTM cell-state update using forget gate \(f_t\), input gate \(i_t\), and candidate state \(\tilde c_t\).
  • LSTMs help with vanishing gradients and long-term dependencies.
Numerical example
In a DNA, protein, or text sequence, the model can use earlier context to interpret later positions.
Annotated Python code

21) k-Means Clustering

Unsupervised · Clustering

Partitions points into \(k\) clusters by alternating between assigning points to nearest centroids and updating centroids.

Core idea

Partitions points into \(k\) clusters by alternating between assigning points to nearest centroids and updating centroids.

When it is useful

Discovering groups without labels

Formula
\[ \min_{\{\mu_k\},\{c_i\}} \sum_{i=1}^n \|x_i-\mu_{c_i}\|^2 \]
Explain the formula
  • \(\mu_{c_i}\) is the centroid of the cluster assigned to point \(i\).
  • The algorithm alternates between assignment and centroid update until convergence.
  • It works best for roughly spherical, similarly sized clusters.
Numerical example
If points [1,2,10,11] are clustered into \(k=2\), the centroids converge near 1.5 and 10.5.
Annotated Python code

22) Hierarchical Clustering

Unsupervised · Clustering

Builds a dendrogram by repeatedly merging the closest clusters under a linkage rule.

Core idea

Builds a dendrogram by repeatedly merging the closest clusters under a linkage rule.

When it is useful

Discovering groups without labels

Formula
\[ d_{\text{avg}}(A,B)=\frac{1}{|A||B|}\sum_{x\in A}\sum_{y\in B} d(x,y) \]
Explain the formula
  • The example formula is average linkage distance between clusters \(A\) and \(B\).
  • Other common linkage rules are single, complete, and Ward.
  • Unlike k-means, hierarchical clustering gives a full tree of cluster relationships.
Numerical example
If clusters A={1,2} and B={8,9}, the average pairwise distance is large, so they merge late in the dendrogram.
Annotated Python code

23) DBSCAN

Unsupervised · Density-Based Clustering

Finds dense regions of points and labels sparse points as noise. It can recover irregularly shaped clusters.

Core idea

Finds dense regions of points and labels sparse points as noise. It can recover irregularly shaped clusters.

When it is useful

Finding irregular clusters and noise points

Formula
\[ N_\varepsilon(x)=\{x_i : d(x_i,x)\le \varepsilon\} \]
Explain the formula
  • A point is a core point if its \(\varepsilon\)-neighborhood contains at least min_samples points.
  • Clusters grow by connecting density-reachable points.
  • Noise points are labeled as outliers instead of being forced into a cluster.
Numerical example
If a point has 6 neighbors within radius 0.5 and min_samples=5, it is a core point and can seed a cluster.
Annotated Python code

24) Gaussian Mixture Model (GMM)

Unsupervised · Probabilistic Clustering

Represents the data distribution as a weighted mixture of Gaussian components, giving soft cluster memberships.

Core idea

Represents the data distribution as a weighted mixture of Gaussian components, giving soft cluster memberships.

When it is useful

Soft cluster membership and density modeling

Formula
\[ p(x)=\sum_{k=1}^K \pi_k \,\mathcal{N}(x\mid \mu_k,\Sigma_k) \]
Explain the formula
  • \(\pi_k\) is the mixture weight of component \(k\), with \(\sum_k \pi_k=1\).
  • Each component has its own mean \(\mu_k\) and covariance \(\Sigma_k\).
  • EM alternates between computing responsibilities and updating parameters.
Numerical example
A point may belong 70% to cluster 1 and 30% to cluster 2 instead of being assigned hard membership.
Annotated Python code

25) Principal Component Analysis (PCA)

Unsupervised · Dimensionality Reduction

Rotates the data into orthogonal directions that capture maximum variance, often used for compression and visualization.

Core idea

Rotates the data into orthogonal directions that capture maximum variance, often used for compression and visualization.

When it is useful

Compression and visualization

Formula
\[ \Sigma = \frac{1}{n-1}X^\top X,\qquad \Sigma v_j = \lambda_j v_j \] \[ Z = XW \]
Explain the formula
  • The covariance matrix \(\Sigma\) summarizes feature variability and correlation.
  • Eigenvectors \(v_j\) give principal directions; eigenvalues \(\lambda_j\) give explained variance.
  • The projected coordinates \(Z\) are the lower-dimensional representation.
Numerical example
If most variance lies along a diagonal line in 2D, one principal component may explain nearly all of the variability.
Annotated Python code

26) Nonnegative Matrix Factorization (NMF)

Unsupervised · Parts-Based Representation

Factorizes a nonnegative matrix into two lower-rank nonnegative matrices, often yielding interpretable parts or latent topics.

Core idea

Factorizes a nonnegative matrix into two lower-rank nonnegative matrices, often yielding interpretable parts or latent topics.

When it is useful

Interpretable latent nonnegative factors

Formula
\[ X \approx WH,\qquad W\ge 0,\ H\ge 0 \]
Explain the formula
  • \(X\) is the original nonnegative matrix, such as counts or intensities.
  • \(W\) contains sample-by-factor weights and \(H\) contains factor-by-feature weights.
  • The nonnegativity constraint often makes the decomposition easier to interpret than PCA.
Numerical example
A gene-expression matrix can be decomposed into a few latent programs, each with nonnegative activity across samples and genes.
Annotated Python code

27) Hidden Markov Model (HMM)

Probabilistic · Sequence Model

Models sequences where an unobserved state process emits the observed data at each step.

Core idea

Models sequences where an unobserved state process emits the observed data at each step.

When it is useful

Latent states behind sequential observations

Formula
\[ p(z_{1:T},x_{1:T}) = p(z_1)\prod_{t=2}^T p(z_t\mid z_{t-1}) \prod_{t=1}^T p(x_t\mid z_t) \]
Explain the formula
  • \(z_t\) is the hidden state at time \(t\), and \(x_t\) is the observed symbol or vector.

30) Autoencoder

Core idea

Learns to compress data into a latent code and reconstruct it, often used for representation learning, denoising, or anomaly detection.

When it is useful

Compression, denoising, latent features

Formula
\[ z = f_{\text{enc}}(x),\qquad \hat{x}=f_{\text{dec}}(z) \] \[ \mathcal{L}=\|x-\hat{x}\|_2^2 \]
Explain the formula
  • The encoder maps the input to a compact latent representation \(z\).
  • The decoder tries to reconstruct the original input from \(z\).
  • If the latent bottleneck is small, the model is forced to learn efficient structure.
Numerical example
A 10-dimensional vector may be compressed into 2 latent values and then reconstructed. Large reconstruction error can indicate an outlier.
